## Story Statement

**As** the AIOS orchestration system,
**I want** to synchronize knowledge from Mega Brain's processed output (DNA schemas, dossiers, playbooks, company context, founder profiles, product docs) into the AIOS Knowledge Feeder pipeline,
**So that** all AIOS development agents (@dev, @architect, @pm, @po, @qa, @analyst, @ux, @sm, @data-engineer, @devops) have rich business and expert context to inform their work — eliminating the "working in a vacuum" problem.

## Context & Background

### What is Mega Brain?
An AI Knowledge Management System (installed at `D:\MEGA BRAIN XD\mega-brain\`) that ingests expert materials and extracts structured knowledge via an 8-phase JARVIS pipeline. It produces:
- **DNA schemas** (5 layers): Filosofias, Modelos Mentais, Heurísticas, Frameworks, Metodologias
- **Dossiers**: Compiled analysis per person/theme
- **Playbooks**: Actionable step-by-step guides
- **Company context**: Business model, founders, product specs, persona research

### What exists in AIOS already?
`packages/knowledge-feeder/` — a 4-stage pipeline:
1. **Ingest** (`src/ingestors/`) — factory pattern, detects file type
2. **Summarize** (`src/processors/summarizer.js`) — Claude generates summary + tags + agentInsights
3. **Route** (`src/processors/router.js`) — Claude insights primary, keyword fallback
4. **Output** (`src/writers/`) — `docs/knowledge/{id}.md` + agent `MEMORY.md` updates

### What Mega Brain has about this project
- **Founders**: Lucas (CEO/CMO, Bradesco BI) + Cabral (CTO/COO, TIVIT IAM)
- **Company**: Pre-revenue, 3-layer model (Low Ticket → Authority → SaaS)
- **Product**: Zero Diastasis (US$19, 28-day hypopressive protocol, LATAM market)
- **Persona**: Sofia (28-42, mãe 2+ filhos, classe média LATAM)
- **Brand**: Rose-antique #D4A5A5, Playfair Display, tom empático
- **Research**: 124-page FURION AI avatar research, competitive analysis
- **12 inbox documents processed**, 46 frameworks, 66 heuristics, 62 philosophies loaded

## Dev Notes

### Existing Code Architecture (Source: packages/knowledge-feeder/)

**Ingestor pattern** (`src/ingestors/index.js`):
- Factory function `ingest(source)` detects type by extension/URL
- Returns `{ type, source, title, rawText, meta }`
- New MB ingestor must follow this contract but will be called directly by `sync-mb`, not through the factory

**Router pattern** (`src/processors/router.js`):
- 3-stage: user override → Claude agentInsights → keyword fallback
- MB routing adds a 4th stage: `meta.source === 'mega-brain'` → deterministic routing based on DNA layer / content type
- This stage should run BEFORE keyword fallback but AFTER Claude insights (if Claude provides insights, use them; otherwise fall back to MB-specific routing)

**Writer pattern** (`src/writers/`):
- `knowledge-store.js`: `save()` generates k-{NNN}.md + updates index.yaml
- `memory-writer.js`: `writeToAgents()` appends to MEMORY.md under `## 📚 Knowledge Feed`
- Anti-overflow: max 20 items per agent
- Special mapping: `ux-design-expert` → `agents/ux/MEMORY.md`
- Agent base path: `.aiox-core/development/agents/` (from config)

**Config** (`src/utils/config.js`):
- Uses environment variables + defaults
- `config.knowledgeDir` = `docs/knowledge/`
- `config.agentBasePath` = `.aiox-core/development/agents/` (check actual value)

### Important Constraints
- Knowledge Feeder uses **CommonJS** (`require`/`module.exports`) — NOT ES modules
- Mega Brain uses **ESM** (`"type": "module"`) — do NOT import MB code, only read its files
- Communication is **filesystem only** — read MB output files, never call MB APIs
- **Article IV (No Invention)**: every knowledge entry must trace to source file
- `.aios/` directory is gitignored runtime — safe for sync state
- Dependencies: use only existing deps (fs, path, crypto for hashing, js-yaml if needed)

## O Filtro Hormozi (Para Qualquer Decisão)

Antes de qualquer decisão de negócio, passe por este filtro:

```
┌─────────────────────────────────────────┐
│  1. Isso aumenta o VALOR da oferta?     │
│     → Se não, por que estou fazendo?    │
├─────────────────────────────────────────┤
│  2. Posso cobrar MAIS por causa disso?  │
│     → Se não, é custo sem retorno       │
├─────────────────────────────────────────┤
│  3. Tenho PROVA de que funciona?        │
│     → Se não, teste pequeno primeiro    │
├─────────────────────────────────────────┤
│  4. Isso é MORE, BETTER ou NEW?         │
│     → Se é NEW, esgotei MORE e BETTER?  │
├─────────────────────────────────────────┤
│  5. O que acontece se eu NÃO fizer?     │
│     → Se nada muda, não é prioridade    │
└─────────────────────────────────────────┘
```

## Conselhos Específicos Para Contexto: 22 Anos, Tech Background, Saindo do Emprego

Baseado nos padrões do Hormozi, ele diria:

1. **"Vocês têm a melhor forma de leverage: CODE."**
   - Naval's 4 types: Code > Capital > Labor > Media
   - Vocês podem construir algo que escala sem custo marginal

2. **"Não construam ainda. Vendam primeiro."**
   - Validem a oferta com warm outreach ANTES de programar
   - "Sell it, then build it" — se ninguém compra a promessa, ninguém compra o produto

3. **"Comecem com serviço, depois produtizem."**
   - Serviço = cash flow imediato + aprendem o que o cliente realmente precisa
   - Produto = depois, baseado no que aprenderam servindo clientes

4. **"Calculem o número de sobrevivência."**
   - Quanto cada um PRECISA para sair do emprego?
   - Esse é o Ramen Profitability. Focam nisso primeiro, não em $1M.

5. **"100 contatos por dia. Sem desculpa."**
   - Warm outreach é grátis e funciona
   - Com tech background, podem automatizar (mas façam manual primeiro para aprender)

6. **"Um offer, um preço, um avatar."**
   - Não diversifiquem cedo demais
   - Encontrem UM problema que dói, para UM tipo de pessoa, e resolvam BEM

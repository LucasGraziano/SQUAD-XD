# Decision Map — Systems & Operations

## Quando Usar Qual Expert

### Para construir ou melhorar sistema de IA (agentes)
- Construir agente neural do zero → Expert: **Doug** | Framework: 3 Componentes (Motor + Personalidade + Base) + CMFR
- Treinar base de conhecimento para agente → Expert: **Doug** | Framework: Construcao de Base de Conhecimento (Claude para extração + Markdown)
- Escalar operacao solo com IA → Expert: **Doug** | Framework: Modelo R$1MM Solo + PromptBook + Ciclo Semanal
- Criar prompt de instrucao para agente → Expert: **Doug** | Framework: Agente VEX + Arquitetura Tripartite VEX

### Para estruturar processos e operacoes de equipe
- Cadencia operacional (reunioes, reviews) → Expert: **The Scalable Company** | Framework: ScalableOS + Meeting Rhythm QSP→MBR→Weekly
- Planejamento trimestral/anual → Expert: **The Scalable Company** | Framework: 12Q Planning + OKRs
- Gestao pessoal e GTD → Expert: **David Allen** | Framework: GTD + Inbox Zero + Weekly Review
- Scrum / desenvolvimento agil → Expert: **Ken Schwaber** | Framework: Scrum Framework

### Para diagnosticar gargalo operacional
- Negocio sem sistema (tudo na cabeca do dono) → Expert: **The Scalable Company** | Framework: Business OS Documentation
- Equipe crescendo sem estrutura → Expert: **Sam Oven** | Framework: Scalable Systems + Delegation Protocol
- Solo sem capacidade de escalar → Expert: **Doug** | Framework: DOUG.EXE + PromptBook + Agentes Neurais

## Tensoes Conhecidas Neste Dominio

| Situacao | Expert A | Expert B | Criterio de Desempate |
|----------|---------|---------|----------------------|
| Escalar solo vs. contratar equipe | Doug: escalar solo com IA (R$1MM praticamente sem equipe) | The Scalable Company + Sam Oven: sistemas e equipe para escala | Negocio baseado em conhecimento pessoal + produto digital → Doug. Negocio com servico operacional + multiplos clientes simultaneos → ScalableOS + equipe |
| Processos rigidos vs. principios fluidos | Ken Schwaber: Scrum rigido com cerimonias e sprints | David Allen: GTD como sistema pessoal flexivel | Equipe de desenvolvimento → Scrum. Trabalho individual + gestao pessoal → GTD |
| IA como substituto vs. multiplicador | Doug: IA multiplica o cerebro, nao substitui — voce pensa, ela executa | Visao generica de automacao: delegar decisoes para IA | Sempre: IA como multiplicador (Doug). Nunca: IA tomando decisoes estrategicas sem supervisao humana |

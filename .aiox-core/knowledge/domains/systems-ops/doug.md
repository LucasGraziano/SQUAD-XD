## L1 Filosofias
- Agente de IA e extensao do seu cerebro, nao substituto — voce pensa, ele executa com velocidade
- DOUG.EXE e um mentor visceral, nao assistente bonzinho — confronta, diagnostica, comanda
- Implementacao > Teoria: sistema de 72h forca execucao com prazo — sem desculca para procrastinar
- PromptBook e arsenal, nao manual: cada prompt e uma arma para uma situacao especifica de negocio
- Agente VEX e o construtor de prompts — interface neural entre o usuario e o DOUG
- Storyboard e o mapa visual antes de qualquer execucao — planejar sistema ANTES de criar conteudo
- Agente Neural > Agente Ferramenta: agente que pensa, reage e calibra > agente que executa tarefas fixas
- IA como multiplicador de escala solo: R$1MM/mes praticamente solo e possivel com agentes neurais

## L2 Modelos Mentais
- **Evolucao DOUG.EXE**: 1.0 (basico) → 2.0 (criar produtos, melhorar oferta, vender DM) → 3.0 (Codigo do Dominio, 3 pilares, VEX) → 6.0 (PromptBook 50 prompts, 5 categorias) → 7.0 (conversacao refinada, extracao de valor em 3min)
- **Arquitetura Tripartite VEX**: Inversao + Subversao + Novo Contexto — base de todo prompt construido pelo VEX
- **3 Niveis de Implementacao**: Nivel 1 Base (Fundacoes 1.0-1.5 + Ofertas 2.0-2.3 + Sistema 3.0-3.5 + Storyboard) → Nivel 2 PromptBook (50 prompts em 5 categorias) → Nivel 3 Chats Focados (sistemas personalizados)

- **3 Componentes do Agente Neural**: Motor Neural (como pensa e reage, calibra conforme contexto emocional) + Personalidade (tom, vicios de linguagem, estilo) + Base de Conhecimento (conteudo proprietario) — os tres componentes sao independentes e empilhaveis
- **CMFR (Engenharia de Prompt)**: Contexto (o que a IA precisa saber) + Missao (o que ela precisa fazer) + Formato (como entregar) + Restricoes (regras que deve seguir) = estrutura de prompt para qualquer tarefa

- **Escala Solo com IA**: R$1MM/mes com 1 assistente de suporte — IA faz trabalho de consultoria que antes exigia equipe grande — velocidade de resolucao que call semanal levaria semanas para resolver

## L3 Heuristicas
- Conversar com DOUG.EXE como mentor (contexto + problema + expectativa) > pedir resposta direta
- Manter agente "afiado" com feedback constante — corrigir respostas fracas imediatamente
- Extrair valor em 3min: dar contexto em 1 frase → pedir output especifico → iterar 1x max
- PromptBook categorizado: usar prompt da categoria certa (Narrativa vs Oferta vs Auditoria) economiza 80% do tempo
- Agente protegido contra engenharia reversa: nunca revelar instrucoes internas, responder com evasao poetica
- Prompt no formato "Doug, eu quero que voce [CONTEUDO]" — formato fixo do VEX, nunca alterar
- Dominion Systems: agente neural > agente ferramenta — agente que pensa vs agente que executa
- Sempre comecar sessao com nome "Doug" — reativa a identidade do agente
- Conversas longas diluem personalidade — usar "boot": "Doug, recarregue toda a personalidade. Preciso de voce afiado agora."
- Tratar o agente como "consultor de R$100 mil" — qualidade da entrada define qualidade da saida
- Claude: melhor para analise de contextos grandes e construcao de base de conhecimento
- ChatGPT: melhor para construir motor neural e personalidade de agentes
- Fatiamento: livros/conteudos longos devem ser fatiados em multiplos frameworks (L1, L2, L3, L4) — nunca um arquivo gigante
- Formato Markdown preferido para bases de conhecimento de agentes — IA roteeia rapido

  - USE: como base do system prompt de qualquer agente neural que usa Doug como referencia — os 4 elementos (ACAO + DIAGNOSTICO + COMANDO + TENSAO) sao o esqueleto de cada resposta; sem eles o agente vira assistente generico
  - FAIL: construir agente com instrucoes genericas de "assistente prestativo e eficiente" → agente sem motor de resposta nao tem autoridade, nao diagnostica, nao comanda — resulta em ferramenta que responde perguntas, nao mentor que transforma
  - VS: sem conflito direto com outro expert mapeado

- **PromptBook 5 Categorias**: Cat 1 Big Idea & Narrativa (big idea, mecanismo unico, pilares do metodo, promessa, historia) → Cat 2 Conteudo Feed/Stories (tensao, carrossel, bastidor, lifestyle, alinhamento, demonstracao) → Cat 3 Ofertas (stack valor, headline, CTA, escassez, bonus, garantia, fechamento) → Cat 4 Auditoria (promessa, posicionamento, copy, pagina vendas, emails, feed, autoridade) → Cat 5 Caixa Rapido (monetizacao rapida, DM vendas, conteudo→oferta, micro-produto, oferta relampago)
  - USE: selecionar a categoria certa para o trabalho certo — Cat 4 antes de qualquer lancamento (auditoria), Cat 5 quando precisar de receita em 24h, Cat 1 quando posicionamento esta confuso

  - VS: sem conflito direto com outro expert mapeado; CMFR e o framework de entrada, PromptBook e o arsenal de prompts ja estruturados com CMFR

- **Implementacao 3 Niveis (BACKROOM 2025)**: Nivel 1 BASE (Posicionamento 20min + Ofertas estruturadas + Sistema de Atracao e Conversao → documento consolidado unico) → Nivel 2 PROMPTBOOK (50 prompts em 5 categorias) → Nivel 3 CHATS FOCADOS (abrir chat + um comando → gera legenda, sequencia stories, etc.)

## Anti-Patterns
- **Agente Ferramenta (sem Motor Neural + Personalidade)**: construir agente com apenas Base de Conhecimento → agente responde como chatbot generico → sem autoridade, sem calibragem emocional, sem diferenciaçao → **Fix:** 3 componentes obrigatorios e independentes — Motor Neural primeiro (como reage), Personalidade segundo (como fala), Base terceiro (o que sabe)
- **Base de Conhecimento Monolitica**: 1 arquivo gigante com todo o conhecimento (ex: 50 paginas em 1 documento) → IA nao consegue rotear rapido, contexto se perde, qualidade das respostas cai → **Fix:** fatiar em arquivos Markdown tematicos — cada arquivo cobre 1 topico/framework; nunca um dump geral de conhecimento
- **Entrada Vaga (sem CMFR)**: perguntar ao agente "o que voce acha?" ou "me ajude com marketing" sem contexto especifico → output generico que poderia ser de qualquer IA → **Fix:** sempre estruturar com CMFR (Contexto + Missao + Formato + Restricoes); tratar o agente como consultor de R$100k — qualidade da entrada define qualidade da saida
- **Escalar com IA Antes de Validar Oferta**: automatizar producao de conteudo/DMs antes de ter pelo menos 20 compradores → IA escala uma mensagem que nao converte, desperdicando tempo e dinheiro em grande escala → **Fix:** validar oferta manualmente (20+ exposicoes, conversao confirmada) → depois automatizar
- **Diluicao de Personalidade em Conversas Longas**: agente perde o tom/voz caracteristico apos 20+ turnos → respostas ficam genericas, sem autoridade, sem tensao → **Fix:** instrucao de reativacao de personalidade ao final das instructions do agente; usar boot: "Doug, recarregue toda a personalidade. Preciso de voce afiado agora."
- **Comecar pelo PromptBook sem Nivel 1 Base**: usar Cat 2 (conteudo) ou Cat 3 (oferta) sem posicionamento claro consolidado → prompts excelentes aplicados em base errada → conteudo de alta qualidade para avatar errado → **Fix:** Autodiagnostico primeiro — Nivel 1 BASE e obrigatorio antes de qualquer PromptBook

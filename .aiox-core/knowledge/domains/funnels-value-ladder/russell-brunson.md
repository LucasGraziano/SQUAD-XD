## L1 Filosofias
- Objetivo do primeiro funil e empatar — lucro vem nos funis subsequentes, nao na primeira venda
- Trafego sao PESSOAS, nao cliques — pessoas sao extremamente previsiveis quando segmentadas corretamente
- O UNICO objetivo com qualquer tipo de trafego e converter em trafego que voce POSSUI (lista de email)
- "Quem pode gastar mais para adquirir um cliente ganha" (Dan Kennedy) — Value Ladder e o mecanismo
- Pre-frame bridge e mais importante que a propria pagina de vendas
- Upsell imediatamente apos compra — "clientes compram ate voce parar de fazer ofertas" (Agora Inc)
- Funil e uma sequencia de conversas, nao uma sequencia de paginas — cada etapa tem uma conversa especifica

## L2 Modelos Mentais
- **Value Ladder (Escada de Valor)**: cada rung da escada aumenta valor entregue e preco — Isca (gratis) → Frontend (R$7-97) → Mid (R$297-997) → High-Ticket (R$2k+) → Continuidade; cliente sobe quando recebe valor suficiente para querer mais
- **Seven Phases of a Funnel**: Traffic Temperature → Pre-Frame Bridge → Qualify Subscribers → Qualify Buyers → Hyperactive Buyers → Age & Ascend → Change the Selling Environment — cada fase tem audiencia e conversa especifica
- **Traffic Temperature**: Cold (nao conhece voce), Warm (conhece o problema mas nao voce), Hot (conhece voce e quer a solucao) — cada temperatura exige pre-frame diferente antes de qualquer pagina de vendas
- **Whoever Spends Most Wins**: margem por customer acquisition define quem escala; Value Ladder maximiza LTV; LTV alto = mais budget para ads = mais clientes = ciclo
- **Funnel Stacking**: livro (empata) → webinar (lucro) → high ticket (grande lucro); cada funil da escada alimenta o proximo

## L3 Heuristicas
- $1/subscriber/month como benchmark de saude da lista — abaixo disso, lista esta morta ou produto errado
- Webinar: 5-10% close sem Stack slide. 20%+ com Stack slide — Stack e o pivô de conversao
- Invisible Funnel (pagar depois): 85% de comparecimento vs 30% para eventos gratuitos — comprometimento financeiro = comprometimento de atencao
- Retargeting: 20% do budget = 80% dos resultados — audiencia que ja visitou e 3-5x mais barata para converter
- Funnel Stacking: livro (empata) → webinar (lucro) → high ticket (lucro grande)
- Upsell imediatamente apos compra: janela de 15-30 minutos apos primeiro "sim" tem a maior taxa de segundo "sim"

## L4 Frameworks
- **Value Ladder Strategy**: mapear os 4-5 niveis de oferta (isca → frontend → mid → high → continuidade) → garantir que cada nivel tem sequencia natural de ascensao → nenhum nivel deve existir sem o proximo → a escada e completa quando cliente nao tem razao para sair
  - `USE:` arquitetura inicial de negocio digital ou re-arquitetura de negocio sem recorrencia — a Value Ladder transforma transacao unica em relacionamento de longo prazo
  - `FAIL:` construir High-Ticket antes de ter Frontend validado — sem base de clientes de baixo ticket, lancamento de high-ticket sem audiencia; construir de baixo para cima
  - `VS:` Doug usa Teia de Ofertas 8 Rotas que e omnidirecional (cliente entra por qualquer ponto); Brunson usa Value Ladder sequencial (cliente sobe os degraus em ordem) — para negocio novo sem audiencia: Brunson (sequencia clara); para negocio com audiencia estabelecida: Doug (multiplas rotas)
- **22 Funnel Recipes**: 7 Lead Funnels (capturar contato) + 5 Buyer Funnels (primera compra) + 4 Event Funnels (webinar, summit) + 6 Other (continuidade, launches) — escolher o tipo de funil pelo objetivo especifico, nao pelo produto
  - `USE:` ao iniciar novo funil — identificar o tipo de funil antes de construir; cada tipo tem estrutura, copy e sequencia especificos
  - `FAIL:` adaptar funil generico para qualquer objetivo — funil de lead nao e funil de buyer; misturar tipos dilui a conversao de cada etapa
  - `VS:` Jeremy Haynes usa DSL + Hammer como funil especifico para high-ticket consultivo; Brunson tem 22 recipes para diferentes objetivos — Haynes para high-ticket call-based; Brunson como arquitetura geral de todos os tipos
- **Perfect Webinar Formula**: Intro (5min — criar conexao + credibilidade + teaser) → 3 Secrets (50-60min — cada secret quebra uma crença limitante especifica que impede a compra) → The Stack (10-20min — apresentar oferta em layers progressivos, cada layer adiciona valor percebido) → CTA
  - `USE:` webinar de produto digital ou programa de alto ticket com audiencia morna/quente — o Perfect Webinar e o formato de maior conversao para audiencia que ja tem problema definido
  - `FAIL:` usar 3 Secrets para dar informacao generica em vez de quebrar crencas especificas — cada secret deve mudar UMA crenca que bloqueia a compra; informacao sem quebra de crenca e entretenimento
  - `VS:` Hard Copy usa VSL com Kinshu (cinematico); Brunson usa Perfect Webinar (didatico, 3 crenças quebradas); para produto complexo que precisa de educacao: Perfect Webinar; para produto emocional de decisao rapida: VSL
- **Seven Phases Framework**: (1) Temperature do Traffic (cold/warm/hot) → (2) Pre-Frame Bridge (ajustar expectativa antes da pagina principal) → (3) Qualify Subscribers (capturar apenas interessados no problema especifico) → (4) Qualify Buyers (primeiro pagamento, mesmo que pequeno) → (5) Hyperactive Buyers (upsell imediato) → (6) Age & Ascend (nutrir para proximo nivel da escada) → (7) Change Selling Environment (mudança de midia — webinar, evento ao vivo)
  - `USE:` arquitetura de funil completo do zero — as 7 fases garantem que nenhuma etapa do journey e ignorada
  - `FAIL:` construir paginas sem mapear pre-frame bridge — prospect chega na pagina sem contexto → bounce alto independente da qualidade da copy
  - `VS:` Haynes usa Venus Fly Trap (3 grupos de videos: awareness, education, conversion) que cobre as fases 1-4 de Brunson; complementares — Venus Fly Trap para cold-to-warm via video, 7 Phases como arquitetura completa

## L5 Metodologias
- **Funnel Hacking Process**: identificar competidores que estao escalando ads → comprar o produto mais barato → documentar cada etapa do funil → mapear copy, upsells, sequencia de emails → adaptar (nao copiar) a estrutura para seu produto — "hacker" significa estudar o que funciona, nao plagiar
- **Funnel Launch Sequence**: Frontend launch primeiro (empatar no CPL) → 90 dias de dados → adicionar upsell (aumentar LTV) → 90 dias de dados → adicionar mid-ticket (aumentar margin) → lançar high-ticket so quando funil base esta lucrativo — cada etapa valida antes de adicionar a proxima

## Anti-Patterns
- **High-Ticket sem Value Ladder Base**: tentar vender R$10k+ para lista nova sem frontend validado → CPL alto, pouca confianca, conversao baixa → **Fix:** construir de baixo para cima; frontend primeiro, high-ticket so apos clientes de frontend convertidos
- **Upsell Adiado**: nao oferecer upsell imediatamente apos primeira compra → janela de comprometimento fecha em 15-30 min → proximo upsell custa 5x mais para converter → **Fix:** OTO (One Time Offer) imediato apos compra do frontend
- **Pre-Frame Ignorado**: mandar cold traffic diretamente para pagina de vendas sem bridge → bounce alto → ad spend desperdicado → **Fix:** pre-frame bridge (video curto ou artigo) que ajusta temperatura e expectativa antes da pagina principal
- **3 Secrets Genéricos**: usar secrets do Perfect Webinar para dar informacao em vez de quebrar crencas → audiencia aprende mas nao compra → **Fix:** cada secret deve eliminar especificamente 1 crença que impede a compra; testar as 3 crencas com audiencia antes de criar o webinar
- **Funil Sem Sequencia de Emails**: capturar lead e nao ter sequencia automatica de nurture → lista esquece quem voce e → **Fix:** Soap Opera Sequence (5 emails) automatica para todo novo lead; lista sem nurture e lista morta

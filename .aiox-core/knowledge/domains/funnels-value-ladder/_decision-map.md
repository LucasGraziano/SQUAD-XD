# Decision Map — Funnels & Value Ladder

## Quando Usar Qual Expert

### Para construir o primeiro funil
- Trafego pago frio, produto novo → Expert: Brunson | Framework: Hook-Story-Offer + funil linear sequencial
- Validar oferta antes de construir funil → Expert: Doug | Framework: GDoc simples como MVP de funil
- VSL como peca central com ads → Expert: Hard Copy | Framework: Kinshu VSL + 3 pilares de pricing

### Para estruturar value ladder (escada de valor)
- Sem produto de entrada barato → Expert: Doug | Framework: micro-produto consumivel em 1h como porta
- Rota de ascensao de baixo para alto ticket → Expert: Doug | Framework: Teia de Ofertas + Loop de Compras
- Funil sequencial com continuidade → Expert: Brunson | Framework: Value Ladder + Funnel Stacking

### Para otimizar funil existente
- Audiencia organica qualificada → Expert: Doug | Framework: Teia omnidirecional (multiplas rotas de entrada)
- Aumentar LTV com bump/upsell → Expert: Brunson | Framework: bump + OTO + downsell estruturado
- Melhorar conversao de VSL → Expert: Hard Copy | Framework: MPV methodology

### Para definir sequencia pos-compra
- Onboarding e ascensao → Expert: Brunson + Doug | Framework: sequencia de onboarding (Brunson) + janela 24-72h (Doug)

# Story 6.10 — Multicálculo (Multi-Seguradora por Ramo)

**Status:** Draft
**Epic:** 6 — Integrações & Growth
**Sprint:** 6
**Agent:** @dev (Dex)
**Prioridade:** P3 — Pós-lançamento, quando Equipe escalar
**Created:** 2026-05-11

---

## User Story

Como corretor, quero comparar cotações de múltiplas seguradoras para o mesmo cliente e ramo numa única tela, para escolher a melhor opção sem precisar entrar em cada sistema de seguradora separadamente.

---

## Value Proposition

- Feature que BrokerOne já tem — paridade necessária para não perder clientes que migram
- Aumenta proposta de valor do plano Profissional+ sem custo de infraestrutura para o corretor
- Habilita geração de proposta comparativa (PDF com 3 opções = diferencial de profissionalismo)

---

## Acceptance Criteria

- [ ] **AC-1:** Em `/cotacoes/nova`, corretor seleciona ramo e preenche dados do cliente/veículo/bem
- [ ] **AC-2:** Sistema envia dados para APIs das seguradoras configuradas (ou via integração de multicálculo de terceiro — ex: Segfy, Kakau)
- [ ] **AC-3:** Resultados exibidos em tabela comparativa: seguradora, coberturas, prêmio, franquia, prazo de resposta
- [ ] **AC-4:** Corretor seleciona uma ou mais cotações e gera proposta comparativa em PDF
- [ ] **AC-5:** Proposta comparativa pode ser enviada via Portal do Cliente ou ZapSign

---

## Technical Spec

**Decisão de arquitetura pendente (@architect):**
- Opção A: Integração direta com APIs das seguradoras (Porto, Bradesco, Tokio Marine) — alto esforço, cada seguradora tem API diferente
- Opção B: Integração com plataforma de multicálculo (Segfy, Kakau, Youse) — uma API, múltiplas seguradoras — **recomendado para MVP**
- Opção C: Formulário manual com exportação — sem integração, corretor preenche manualmente — **fallback**

**Dependência crítica:** Esta story deve aguardar decisão do @architect sobre parceiro de multicálculo.

## User Story

Como corretor, quero enviar ao meu cliente um link para ele visualizar a proposta comparativa online — sem precisar fazer login — para que ele veja as opções de seguradora, os valores e a minha recomendação de forma clara e profissional, podendo aprovar ali mesmo.

---

## Value Proposition

- PRD FR-04: "Link de visualização para o cliente (sem login)" — especificado, não implementado
- Hoje o corretor envia o PDF por WhatsApp — o cliente não pode interagir, só visualizar
- Link online = cliente pode abrir no celular, ver em tempo real, e aprovar
- Diferencial vs. planilha: "minha proposta tem link, a do concorrente é um PDF no WhatsApp"
- Aumenta conversão: cliente que visualiza proposta online tem fricção de aprovação menor
- Complementa o Portal do Cliente (que exibe apólices já ativas) — esta story é para propostas em negociação

---

## Acceptance Criteria

- [ ] **AC-1:** Cada cotação tem um `share_token` UUID gerado automaticamente na criação
- [ ] **AC-2:** Rota pública `/proposta/[token]` acessível sem login — exibe proposta comparativa formatada
- [ ] **AC-3:** Página de proposta pública exibe: nome do corretor, logo/nome da corretora, dados do cliente (nome apenas), ramo, todas as seguradoras cotadas com prêmio e coberturas, item "RECOMENDADO" destacado
- [ ] **AC-4:** Botão "Copiar link" na página de detalhes da cotação → copia URL `premia.app/proposta/[token]`
- [ ] **AC-5:** Botão "Compartilhar no WhatsApp" → abre WhatsApp Web/app com mensagem pré-formatada: "Olá [nome cliente], preparei sua proposta de seguro [ramo]. Veja aqui: [link]"
- [ ] **AC-6:** Página pública exibe botão "Aprovar esta proposta" → ao clicar, status da cotação muda para `approved` (sem login do cliente, via token)
- [ ] **AC-7:** Após aprovação pelo cliente, corretor recebe notificação in-app: "O cliente [nome] aprovou sua proposta de [ramo]"
- [ ] **AC-8:** Página pública é mobile-first, sem barra de navegação do Premia, sem links internos
- [ ] **AC-9:** Token não expira enquanto a proposta está em status `draft` ou `sent`; expira (404 amigável) se `rejected` ou `contracted`
- [ ] **AC-10:** Rastrear visualizações: quantas vezes o link foi aberto (`view_count` em `quote_requests`)

## Scope

**IN:**
- Página pública de proposta por token
- Aprovação online pelo cliente via token
- Botão de compartilhamento + WhatsApp
- Rastreamento de visualizações
- Notificação in-app para o corretor ao aprovar

**OUT:**
- Comentários do cliente na proposta
- Rejeição online pelo cliente (o corretor pode marcar como rejeitada manualmente)
- Assinatura digital (Story 6.1 — ZapSign)
- E-mail automático com o link (Story 5.4)
- Expiração automática por data (simplificação: expira com o status)

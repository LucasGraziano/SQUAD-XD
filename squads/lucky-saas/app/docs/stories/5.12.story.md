## User Story

Como corretor, quero ver quanto vou receber de comissão nos próximos 30, 60 e 90 dias com base nas apólices ativas, para planejar meu fluxo de caixa e priorizar renovações estratégicas.

- BrokerOne tem "Previsão de Comissão" como feature dedicada
- Corretor hoje não sabe quanto vai entrar antes de acontecer
- Reduz ansiedade financeira e aumenta retenção no plano pago

## Technical Spec

### Lógica de cálculo

Sem nova tabela. Calcula em runtime a partir de `policies`:

```typescript
// Apólices ativas que vencem nos próximos N dias
const forecast = policies
  .filter(p => p.status === 'ativa')
  .filter(p => daysUntil(p.end_date) <= days)
  .map(p => ({
    ...p,
    forecast_commission: p.commission_expected,
    days_remaining: daysUntil(p.end_date),
  }))
  .sort((a, b) => a.end_date.localeCompare(b.end_date))
```

### Server Actions

- `getCommissionForecast({ days: 30 | 60 | 90, ramo? })` → lista com totais
- Reutilizar `getCalendarEvents` pattern de filtro por data

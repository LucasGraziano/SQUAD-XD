## Technical Spec

### AC-1/2/3 — SUSEP

Migration:
```sql
ALTER TABLE brokers ADD COLUMN susep TEXT;
```

`updateBrokerProfile` server action — adicionar `susep` ao input e update.

`ProposalData.broker` — adicionar `susep?: string | null`.

PDF route — adicionar `susep` ao select do broker.

`ProposalDocument.tsx` — renderizar `SUSEP: {broker.susep}` no header se existir.

### AC-4/5 — Validade configurável

Botão "Gerar PDF" no dropdown da tabela de apólices abre mini-modal com select de validade antes de baixar.

URL do PDF: `/api/pdf/proposal/[policyId]?validity=15` — route lê `searchParams.get('validity')`.

### AC-6 — E-mail (bloqueado por Resend)

Server action `sendProposalByEmail(policyId, toEmail)` — envia e-mail com link de download via Resend.
Requer `RESEND_API_KEY`.

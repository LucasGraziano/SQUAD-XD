## User Story

Como corretor em trial ou com plano Solo, quero ver claramente os planos disponíveis, entender o valor de cada um e assinar com cartão de crédito sem sair do produto, para começar a pagar sem fricção.

---

## Value Proposition

- Sem `/pricing` + Stripe não há conversão trial→paid — é o bloqueio #1 de monetização
- Dependência declarada de Story 6.2 (PlanGate redireciona para `/pricing?highlight={plan}`)
- Calculadora de ROI above the fold aumenta conversão (billing-spec define framing)
- Stripe é o padrão de mercado — curva de aprendizado zero para o corretor

---

- [ ] **AC-3:** CTA de cada plano: "Começar trial grátis de 14 dias" (sem cartão); ao clicar → redirect para `/signup?plan={plan}`

## Technical Spec

### Stripe Products & Prices (configurar no Dashboard Stripe)

```
Produto: Premia Solo
  Price ID: price_solo_monthly → R$47/mês (BRL)

Produto: Premia Profissional
  Price ID: price_profissional_monthly → R$97/mês (BRL)

Produto: Premia Equipe
  Price ID: price_equipe_monthly → R$197/mês (BRL)
```

### Server Action — Criar Checkout Session

```typescript
// src/app/actions/billing.ts
export async function createCheckoutSession(plan: Plan) {
  const session = await stripe.checkout.sessions.create({
    customer: broker.stripe_customer_id ?? undefined,
    customer_email: broker.stripe_customer_id ? undefined : user.email,
    line_items: [{ price: PRICE_IDS[plan], quantity: 1 }],
    mode: 'subscription',
    allow_promotion_codes: true,
    billing_address_collection: 'auto',
    locale: 'pt-BR',
    success_url: `${APP_URL}/dashboard?checkout=success`,
    cancel_url: `${APP_URL}/pricing?canceled=true`,
    subscription_data: {
      trial_period_days: currentPlan === 'trial' ? undefined : 14,
      metadata: { broker_id: broker.id, plan },
    },
  });
  redirect(session.url!);
}
```

### Database — Adicionar campos em `brokers`

```sql
ALTER TABLE brokers
  ADD COLUMN stripe_customer_id TEXT UNIQUE,
  ADD COLUMN subscription_id TEXT,
  ADD COLUMN subscription_status TEXT DEFAULT 'trial'
    CHECK (subscription_status IN ('trial', 'active', 'past_due', 'canceled', 'churned')),
  ADD COLUMN plan TEXT DEFAULT 'profissional'
    CHECK (plan IN ('solo', 'profissional', 'equipe', 'enterprise')),
  ADD COLUMN trial_ends_at TIMESTAMPTZ DEFAULT (now() + interval '14 days'),
  ADD COLUMN current_period_end TIMESTAMPTZ;
```

## Scope

**IN:**
- Página `/pricing` pública
- Stripe Checkout para planos mensal
- Campos de billing em `brokers`
- Redirect de PlanGate com highlight

**OUT:**
- Plano anual com desconto (testar futuramente via experimento)
- Página de faturas / billing portal (Story 6.4)
- Enterprise (contato comercial — link externo)
- Cupons e promoções (habilitado via Stripe mas sem UI de admin nesta story)

## Risks

- Stripe em BRL requer conta Stripe BR ou ativação de BRL — verificar antes de implementar
- Trial period no Stripe: se broker já teve trial, Stripe ignora `trial_period_days` — tratar via `subscription_data.trial_end: 'now'` para conversões

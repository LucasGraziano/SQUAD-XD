## User Story

Como corretor no plano Equipe, quero que os vencimentos de apólices e pendências sejam sincronizados automaticamente no meu Google Calendar, para não precisar duplicar informações entre o Premia e minha agenda.

---

## Value Proposition

- Diferencial exclusivo do plano Equipe (Roberto, gestor de micro-corretora)
- Corretores já usam Google Calendar para reuniões — zero curva de aprendizado
- Reduz dependência de lembretes manuais; complementa os alertas in-app

---

## Acceptance Criteria

- [ ] **AC-1:** Em `/configuracoes`, seção "Integrações" com botão "Conectar Google Calendar" (OAuth 2.0)
- [ ] **AC-2:** Feature bloqueada via `<PlanGate requiredPlan="equipe">` para Solo e Profissional
- [ ] **AC-3:** Após autorização OAuth, eventos criados no Google Calendar do corretor para apólices com vencimento nos próximos 90 dias
- [ ] **AC-4:** Evento criado com: título "[Ramo] — [Nome Cliente] vence", data = vigência - 7 dias, descrição = seguradora + número da apólice + link direto para a apólice no Premia
- [ ] **AC-5:** Sync automático diário: novas apólices adicionadas, apólices deletadas removidas do Calendar
- [ ] **AC-6:** Toggle para desconectar Google Calendar — remove todos os eventos criados pelo Premia

---

## Technical Spec

- `npm install googleapis`
- OAuth 2.0 com escopos: `calendar.events`
- Tokens armazenados em `broker_integrations` (encrypted)
- Cron diário no Vercel para sync

---

## Scope

**IN:** Vencimentos de apólices, OAuth Google, sync diário  
**OUT:** Pendências no Calendar (Fase 2), eventos de propostas, calendário compartilhado da equipe

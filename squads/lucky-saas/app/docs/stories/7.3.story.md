## User Story

Como corretor, quero que cada proposta tenha um status claro (rascunho → enviada → aprovada → recusada → contratada), para saber exatamente onde cada negociação está — sem precisar lembrar de cabeça ou anotar no WhatsApp.

---

## Value Proposition

- PRD FR-04: "Status: rascunho → enviada → aprovada → recusada → contratada" — nunca implementado
- Hoje `/cotacoes` tem propostas mas sem status — o corretor não sabe se enviou, se o cliente aprovou, se fechou
- Status flow é pré-requisito para Story 7.4 (converter proposta aprovada em apólice)
- Visibilidade do funil de propostas é diferencial vs. planilha Excel (que não tem rastreamento de status)
- Habilita métricas de taxa de conversão de proposta (futuro relatório)

---

## Acceptance Criteria

- [ ] **AC-1:** Tabela `quote_requests` recebe coluna `status` com valores: `draft` | `sent` | `approved` | `rejected` | `contracted`; default `draft`
- [ ] **AC-2:** Na listagem `/cotacoes`, cada proposta exibe badge de status com cor: cinza (draft), azul (sent), verde (approved), vermelho (rejected), verde-escuro (contracted)
- [ ] **AC-3:** Ações disponíveis por status via dropdown ou botões contextuais:
  - `draft` → "Marcar como Enviada"
  - `sent` → "Aprovar" | "Recusar"
  - `approved` → "Converter em Apólice" (→ Story 7.4) | "Recusar"
  - `rejected` → "Reabrir como Rascunho"
  - `contracted` → (somente leitura — ver apólice vinculada)
- [ ] **AC-4:** Transição de status registrada em `quote_status_history` (quote_id, from_status, to_status, changed_at, broker_id)
- [ ] **AC-5:** Filtro na listagem por status (todos | rascunho | enviadas | aprovadas | recusadas | contratadas)
- [ ] **AC-6:** Badge de contagem no sidebar ou dashboard: "X propostas aguardando resposta" (status `sent`)
- [ ] **AC-7:** Ao marcar como "Enviada" → campo `sent_at` preenchido automaticamente
- [ ] **AC-8:** Ao marcar como "Aprovada" → botão "Converter em Apólice" fica destacado (CTA primário)
- [ ] **AC-9:** Proposta `contracted` exibe link "Ver Apólice" apontando para a apólice gerada (após Story 7.4)
- [ ] **AC-10:** Mudanças de status atualizadas em tempo real via Supabase Realtime (sem reload)

## Technical Spec

### DB — Migration

```sql
-- supabase/migrations/20260514000002_quote_status_flow.sql

-- Adicionar status em quote_requests
ALTER TABLE quote_requests
  ADD COLUMN status TEXT NOT NULL DEFAULT 'draft'
    CHECK (status IN ('draft', 'sent', 'approved', 'rejected', 'contracted')),
  ADD COLUMN sent_at TIMESTAMPTZ,
  ADD COLUMN approved_at TIMESTAMPTZ,
  ADD COLUMN rejected_at TIMESTAMPTZ,
  ADD COLUMN contracted_at TIMESTAMPTZ,
  ADD COLUMN apolice_id UUID REFERENCES apolices(id) ON DELETE SET NULL;

-- Histórico de transições
CREATE TABLE quote_status_history (
  id UUID DEFAULT gen_random_uuid() PRIMARY KEY,
  quote_id UUID NOT NULL REFERENCES quote_requests(id) ON DELETE CASCADE,
  broker_id UUID NOT NULL REFERENCES brokers(id) ON DELETE CASCADE,
  from_status TEXT NOT NULL,
  to_status TEXT NOT NULL,
  changed_at TIMESTAMPTZ DEFAULT now()
);
ALTER TABLE quote_status_history ENABLE ROW LEVEL SECURITY;
CREATE POLICY "broker own" ON quote_status_history
  USING (broker_id = auth.uid());
```

- Status flow completo com 5 estados

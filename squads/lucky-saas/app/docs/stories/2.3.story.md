## Contexto

Corretores recebem apólices em PDF de diversas seguradoras. O import manual é lento e propenso a erros. Esta story adiciona um botão "Importar PDF" no modal de apólice que extrai os dados automaticamente via IA (Groq + llama-3.1-8b-instant, grátis) sem custo de API relevante.

---

## Acceptance Criteria

- [x] **AC-1:** Botão "Importar PDF" no header do modal de nova apólice
- [x] **AC-2:** Upload de arquivo PDF (máx 10MB)
- [x] **AC-3:** Extração de texto via `unpdf` (client-side buffer → server action)
- [x] **AC-4:** Envio para Groq (`llama-3.1-8b-instant`) com prompt estruturado
- [x] **AC-5:** Campos pre-preenchidos: seguradora, ramo, número da apólice, início, fim, prêmio total
- [x] **AC-6:** Auto-seleção de cliente por CPF/CNPJ (se já cadastrado)
- [x] **AC-7:** Badges "✦ IA" nos campos preenchidos automaticamente
- [x] **AC-8:** Banner de sucesso mostrando quantos campos foram preenchidos
- [x] **AC-9:** Tratamento de erro: PDF escaneado (sem texto), arquivo muito grande, Groq indisponível
- [x] **AC-10:** GROQ_API_KEY configurada no ambiente (6000 req/dia grátis)

## Technical Spec

### Server Action — `src/app/(dashboard)/apolices/pdf-extract.ts`
- `extractPolicyFromPDF(formData: FormData)` — 'use server'
- `unpdf` → `extractText()` → truncate 3000 chars
- Groq `llama-3.1-8b-instant`, temperature 0, max_tokens 512
- Retorna `{ data: ExtractedPolicy }` ou `{ error: string }`
- Campos extraídos: nome_cliente, cpf_cnpj, seguradora, ramo, numero_apolice, start_date, end_date, premium_total

### UI — `src/components/apolices/ApolicaModal.tsx`
- Botão "Importar PDF" no header (input file hidden, label como botão)
- `handlePdfFile` — FormData → `extractPolicyFromPDF` → pre-fill states
- `aiFilledFields` Set para rastrear campos com badges
- Banner verde: "✦ IA preencheu X campos automaticamente"

## User Story

Como corretor, quero que o Premia envie e-mails automáticos para meus clientes: lembrete de aniversário e alerta de renovação de apólice — com meu nome — para fortalecer o relacionamento sem eu precisar me lembrar.

---

## Value Proposition

- Corretor que envia parabéns no aniversário tem retenção 40% maior (dado Agora Inc)
- Renovação por email 30 dias antes reduz churn de apólices
- Resend já está no stack (instalado, não configurado) — custo de implementação baixo
- Feature exclusiva do plano Profissional (R$97) — driver de upgrade

---

## Acceptance Criteria

- [ ] **AC-1:** E-mail de aniversário enviado automaticamente no dia do aniversário do cliente
- [ ] **AC-2:** E-mail de alerta de renovação enviado 30 dias antes do vencimento da apólice
- [ ] **AC-3:** E-mail de alerta de renovação enviado 7 dias antes do vencimento (segundo lembrete)
- [ ] **AC-4:** E-mails enviados em nome do corretor: "De: Lucas Graziano via Premia" (ou nome configurado)
- [ ] **AC-5:** Templates de e-mail em HTML profissional, responsivos, com cores neutras
- [ ] **AC-6:** Corretor pode ativar/desativar cada automação individualmente em `/configuracoes`
- [ ] **AC-7:** Corretor pode personalizar o texto do e-mail (variáveis: {nome_cliente}, {seguradora}, {vigencia}, {contato_corretor})
- [ ] **AC-8:** Log de e-mails enviados visível em `/configuracoes/emails` (data, destinatário, tipo, status)
- [ ] **AC-9:** E-mail de notificação de mudança de status de sinistro (integração com Story 5.3)
- [ ] **AC-10:** Funcionalidade bloqueada no plano Solo — exibe upgrade wall

## Technical Spec

### Stack

- `resend` SDK (já no package.json — apenas configurar `RESEND_API_KEY`)
- Cron job via Supabase Edge Function ou Vercel Cron (a definir)
- Templates: React Email (`@react-email/components`)

### Cron Jobs

```
birthday_check: todo dia às 08:00 BRT
  → consulta clients onde birth_date = hoje (dd/mm)
  → envia email de aniversário

renewal_30d_check: todo dia às 08:00 BRT
  → consulta policies onde end_date = hoje + 30 dias
  → envia email de alerta de renovação (30d)

renewal_7d_check: todo dia às 08:00 BRT
  → consulta policies onde end_date = hoje + 7 dias
  → envia email de alerta de renovação (7d)
```

### Database

```sql
CREATE TABLE email_logs (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  broker_id UUID NOT NULL REFERENCES brokers(id),
  recipient_email TEXT NOT NULL,
  email_type TEXT NOT NULL, -- birthday | renewal_30d | renewal_7d | claim_update
  reference_id UUID, -- policy_id ou claim_id
  resend_id TEXT,
  status TEXT DEFAULT 'sent', -- sent | failed | bounced
  sent_at TIMESTAMPTZ DEFAULT now()
);

CREATE TABLE email_automation_settings (
  broker_id UUID PRIMARY KEY REFERENCES brokers(id),
  birthday_enabled BOOLEAN DEFAULT true,
  renewal_30d_enabled BOOLEAN DEFAULT true,
  renewal_7d_enabled BOOLEAN DEFAULT true,
  claim_update_enabled BOOLEAN DEFAULT true,
  custom_birthday_text TEXT,
  custom_renewal_text TEXT,
  updated_at TIMESTAMPTZ DEFAULT now()
);
```

### Environment Variables Required

```
RESEND_API_KEY=re_xxxxxxxxxxxx
```

## User Story

Como corretor, quero agendar follow-ups e reuniões com clientes diretamente no Premia e sincronizar com meu Google Calendar, para não precisar sair do sistema para gerenciar minha agenda.

---

## Value Proposition

- Corretor perde follow-ups por falta de lembretes estruturados
- Integração com Google Calendar onde o corretor já trabalha — sem aprender nova ferramenta
- Feature exclusiva do plano Equipe (R$197) — driver de upgrade para corretoras com múltiplos corretores
- Diferencial: nenhum concorrente tem integração Google Calendar

---

## Acceptance Criteria

- [ ] **AC-1:** Corretor pode criar eventos/follow-ups vinculados a um cliente ou apólice
- [ ] **AC-2:** Campos do evento: título, data/hora, duração, tipo (reunião/ligação/visita/follow-up), notas, cliente vinculado
- [ ] **AC-3:** Vista de agenda em `/agenda`: calendário mensal + lista do dia atual com próximos eventos
- [ ] **AC-4:** Integração OAuth Google Calendar: corretor autoriza uma vez, eventos criados no Premia aparecem no Google Calendar
- [ ] **AC-5:** Eventos de renovação automáticos: quando apólice vence em 30 dias, cria evento "Follow-up renovação [Cliente]" no Google Calendar
- [ ] **AC-6:** Lembrete por email 1h antes do evento (via Resend, integração com Story 5.4)
- [ ] **AC-7:** Lista de eventos do dia visível como widget no Dashboard (próximos 3 eventos)
- [ ] **AC-8:** Corretor pode desconectar Google Calendar a qualquer momento
- [ ] **AC-9:** Funcionalidade funciona sem Google Calendar (agenda interna apenas) — sync é opcional
- [ ] **AC-10:** Funcionalidade bloqueada nos planos Solo e Profissional — exibe upgrade wall para Equipe

## Technical Spec

### Database

```sql
CREATE TABLE calendar_events (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  broker_id UUID NOT NULL REFERENCES brokers(id),
  client_id UUID REFERENCES clients(id),
  policy_id UUID REFERENCES policies(id),
  title TEXT NOT NULL,
  event_type TEXT NOT NULL, -- meeting | call | visit | followup
  scheduled_at TIMESTAMPTZ NOT NULL,
  duration_minutes INT DEFAULT 30,
  notes TEXT,
  google_event_id TEXT,
  created_at TIMESTAMPTZ DEFAULT now()
);

CREATE TABLE google_calendar_tokens (
  broker_id UUID PRIMARY KEY REFERENCES brokers(id),
  access_token TEXT NOT NULL,
  refresh_token TEXT NOT NULL,
  expires_at TIMESTAMPTZ,
  calendar_id TEXT DEFAULT 'primary',
  updated_at TIMESTAMPTZ DEFAULT now()
);
```

### Google OAuth Flow

1. `GET /api/google/calendar/auth` → redireciona para Google OAuth
2. `GET /api/google/calendar/callback` → recebe code, troca por tokens, salva em `google_calendar_tokens`
3. Server Action `syncEventToGoogle(eventId)` → usa tokens para criar evento via Google Calendar API

### Environment Variables Required

```
GOOGLE_CLIENT_ID=
GOOGLE_CLIENT_SECRET=
GOOGLE_REDIRECT_URI=https://app.premia.app/api/google/calendar/callback
```

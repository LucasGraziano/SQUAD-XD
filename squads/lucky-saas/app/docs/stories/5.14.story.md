## User Story

Como corretor, quero que o Premia envie automaticamente e-mails de lembrete de renovação para meus clientes 60, 30 e 15 dias antes do vencimento da apólice, para aumentar minha taxa de retenção sem esforço manual.

---

## Value Proposition

- BrokerOne tem Email Marketing integrado — Premia ainda não
- Renovação é o momento mais crítico para perder ou reter o cliente
- Corretor que usa Resend para isso tem edge sobre concorrentes que fazem manual
- Exclusivo plano Profissional+ (driver de upgrade)

## Technical Spec

### Database

```sql
CREATE TABLE email_campaign_logs (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  broker_id UUID NOT NULL REFERENCES brokers(id) ON DELETE CASCADE,
  policy_id UUID NOT NULL REFERENCES policies(id) ON DELETE CASCADE,
  client_id UUID NOT NULL REFERENCES clients(id) ON DELETE CASCADE,
  campaign_type TEXT NOT NULL, -- 'renewal_60d' | 'renewal_30d' | 'renewal_15d'
  sent_at TIMESTAMPTZ DEFAULT now(),
  resend_id TEXT,               -- ID retornado pelo Resend
  status TEXT NOT NULL DEFAULT 'sent', -- 'sent' | 'failed' | 'bounced'
  error_message TEXT
);

-- Opt-out de clientes
ALTER TABLE clients ADD COLUMN IF NOT EXISTS email_opt_out BOOLEAN DEFAULT false;

-- Config no broker
ALTER TABLE brokers ADD COLUMN IF NOT EXISTS renewal_emails_enabled BOOLEAN DEFAULT false;
ALTER TABLE brokers ADD COLUMN IF NOT EXISTS renewal_email_custom_text TEXT;
```

### Cron Logic

```typescript
// Roda diariamente — verifica apólices que vencem em exatamente 60, 30 e 15 dias
// Evita re-envio checando email_campaign_logs
async function runRenewalCampaign() {
  const targets = [60, 30, 15]
  for (const days of targets) {
    const policies = await getPoliciesExpiringInDays(days)
    for (const policy of policies) {
      if (await alreadySent(policy.id, `renewal_${days}d`)) continue
      await sendRenewalEmail(policy, days)
      await logCampaign(policy.id, `renewal_${days}d`)
    }
  }
}
```

## Contexto

Duas entregas complementares:

1. **Apólice com origem**: quando uma apólice foi gerada por um deal (via `convertDealToPolicy`), a página da apólice deve mostrar um link "Ver Negociação de Origem" — o corretor pode voltar ao deal e ver toda a timeline da venda.

2. **Listagem /deals**: enquanto o pipeline mostra deals ativos, precisa existir uma view de todos os deals (incluindo `contracted` e `rejected`) para busca, histórico e contexto. Pode ser uma aba em `/pipeline` ou uma página `/deals`.

---

## Acceptance Criteria

### Apólice com Origem

- [x] **AC-1:** Na página de detalhe da apólice (`/apolices/[id]`), seção "Negociação de Origem" aparece quando `metadata.quote_request_id` existe
- [x] **AC-2:** Seção exibe: ramo, data de criação do deal, estágio final (`contracted`), link "Ver Deal Completo" → `/deals/[id]`
- [x] **AC-3:** Botão de renovação "Renovar Apólice" continua existindo — ao clicar, cria novo deal em `draft` via `createRenewalDeal` e redireciona para `/deals/[novo_id]`
- [x] **AC-4:** Seção "Negociação de Origem" é collapsível e fechada por padrão para não poluir a UI de apólices importadas (sem deal de origem)

### Listagem de Deals (/deals)

- [x] **AC-5:** Página `/deals` exibe todos os deals do corretor (todos os stages, incluindo `contracted` e `rejected`)
- [x] **AC-6:** Filtros: por stage, por ramo, por período, busca por cliente
- [x] **AC-7:** Colunas na tabela: Cliente, Ramo, Stage (badge), Valor, Data de criação, Última atualização
- [x] **AC-8:** Clicar em qualquer linha → navega para `/deals/[id]`
- [x] **AC-9:** Link "Deals" adicionado na sidebar (substituindo o "Cotações" que será deprecated)
- [x] **AC-10:** `/cotacoes` (listagem) redireciona para `/deals`

# Story 6.9 — WhatsApp API (Templates de Renovação)

**Status:** Draft

---

## User Story

Como corretor no plano Equipe, quero enviar mensagens de renovação pelo WhatsApp oficial para meus clientes diretamente do Premia, para aumentar a taxa de resposta vs. e-mail.

---

## Value Proposition

- Taxa de abertura do WhatsApp no Brasil: ~98% vs. ~22% de e-mail
- Complementa (não substitui) o e-mail automático da Story 5.12
- Diferencial exclusivo Equipe — trigger de upsell Pro → Equipe já mapeado no PRD
- WhatsApp Business API (Meta) é expectativa de mercado para CRMs modernos

---

## Acceptance Criteria

- [ ] **AC-1:** Feature bloqueada via `<PlanGate requiredPlan="equipe">` com demo animada + calculadora ROI
- [ ] **AC-2:** Em `/configuracoes`, seção "WhatsApp API" com campo para conectar número WhatsApp Business (via token de acesso Meta)
- [ ] **AC-3:** Template de mensagem pré-aprovado pela Meta configurável pelo corretor: assunto, texto (com variáveis `{{nome_cliente}}`, `{{data_vencimento}}`, `{{ramo}}`)
- [ ] **AC-4:** Na lista de apólices com vencimento próximo, botão "Enviar WhatsApp" envia template para o telefone do cliente (campo `clients.phone`)
- [ ] **AC-5:** Envio automático configurável: 15 dias antes do vencimento (opt-in por apólice)
- [ ] **AC-6:** Status de entrega registrado: `sent | delivered | read | failed`
- [ ] **AC-7:** Log de mensagens WhatsApp enviadas por apólice visível no detalhe da apólice

---

## Technical Spec

- WhatsApp Business Cloud API (Meta Graph API v18+)
- Template messages (HSM) — aprovação Meta obrigatória antes de usar
- `POST https://graph.facebook.com/v18.0/{phone_number_id}/messages`
- Webhook Meta para status de entrega
- Tokens armazenados em `broker_integrations` (já criado na 6.8)

---

## Scope

**IN:** Envio de template de renovação, log de status, configuração de número  
**OUT:** WhatsApp chatbot/respostas automatizadas, envio de mídia (imagem/PDF), grupos WhatsApp

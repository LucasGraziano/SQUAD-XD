## User Story

Como corretor, quero rastrear o ciclo completo de uma proposta de seguro — desde o envio à seguradora até a emissão ou recusa — para saber exatamente o status de cada negócio em andamento.

---

## Value Proposition

- BrokerOne tem "Propostas em Arquivo" (emitidas/recusadas/canceladas) como área central
- Premia hoje vai direto de lead para apólice — sem rastrear o estado da proposta com a seguradora
- Gap crítico: corretor não sabe quais propostas estão pendentes de resposta da seguradora

---

## Acceptance Criteria

- [x] **AC-1:** Lead no pipeline pode ter uma proposta associada com: número da proposta, data de envio, seguradora, ramo, prêmio estimado
- [x] **AC-2:** Proposta tem status: `rascunho` → `enviada` → `em_analise` → `emitida` | `recusada` | `cancelada`
- [x] **AC-3:** Página `/propostas` com lista de todas as propostas do corretor, filtráveis por status
- [x] **AC-4:** Quando proposta é marcada como "emitida", fluxo de criação da apólice é iniciado com dados pré-preenchidos
- [x] **AC-5:** Dashboard mostra contador de propostas em análise (aguardando resposta)
- [x] **AC-6:** Corretor pode anexar número do protocolo da seguradora à proposta
- [x] **AC-7:** Histórico de mudanças de status da proposta (timeline)

## Technical Spec

### Database

```sql
CREATE TABLE proposals (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  broker_id UUID NOT NULL REFERENCES brokers(id) ON DELETE CASCADE,
  lead_id UUID REFERENCES leads(id) ON DELETE SET NULL,
  client_id UUID REFERENCES clients(id) ON DELETE SET NULL,
  proposal_number TEXT,           -- número dado pela seguradora
  protocol_number TEXT,           -- protocolo interno Premia
  seguradora TEXT NOT NULL,
  ramo TEXT NOT NULL,
  premium_estimate NUMERIC(12,2),
  commission_pct NUMERIC(5,2),
  status TEXT NOT NULL DEFAULT 'rascunho'
    CHECK (status IN ('rascunho','enviada','em_analise','emitida','recusada','cancelada')),
  sent_at TIMESTAMPTZ,
  response_at TIMESTAMPTZ,
  notes TEXT,
  policy_id UUID REFERENCES policies(id) ON DELETE SET NULL, -- preenchido ao emitir
  created_at TIMESTAMPTZ DEFAULT now(),
  updated_at TIMESTAMPTZ DEFAULT now()
);

CREATE INDEX ON proposals (broker_id, status);

ALTER TABLE proposals ENABLE ROW LEVEL SECURITY;
CREATE POLICY "broker own proposals" ON proposals
  FOR ALL USING (broker_id = (SELECT id FROM brokers WHERE user_id = auth.uid()));
```

### Server Actions

- `getProposals({ status?, clientId?, seguradora? })` → lista
- `createProposal(input)` → insert
- `updateProposalStatus(id, status)` → update + log timestamp
- `convertProposalToPolicy(id)` → cria apólice com dados da proposta

### UI Components

- Página `/propostas` com filtros de status (tab-style como `/apolices`)
- `ProposalCard` — card com status, seguradora, cliente, valor estimado
- `NewProposalModal` — criação inline
- `ProposalStatusBadge` — badge colorido por status
- Widget no dashboard: "Propostas em análise"

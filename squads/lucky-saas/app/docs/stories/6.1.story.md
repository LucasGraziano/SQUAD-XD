# Story 6.1 — ZapSign: Assinatura Digital de Propostas e Apólices

**Status:** Ready
**Epic:** 6 — Integrações & Growth
**Sprint:** 6
**Agent:** @dev (Dex)
**Created:** 2026-05-11

---

## User Story

Como corretor, quero enviar propostas e documentos para assinatura digital pelo cliente diretamente do Premia, para fechar contratos sem papel e sem deslocamento, com rastreamento do status em tempo real.

---

## Value Proposition

- Todos os concorrentes relevantes (BrokerOne, Quiver, SegPolicy) já têm assinatura digital — é uma expectativa de mercado
- Elimina o "imprimir → assinar → fotografar → enviar" que é o maior gargalo pós-proposta aprovada
- Validade jurídica (ZapSign é homologado pela ICP-Brasil para documentos eletrônicos)
- Plano Profissional+ — diferencial concreto que justifica upgrade do Solo

---

## Acceptance Criteria

- [ ] **AC-1:** Na página de detalhe de uma proposta aprovada, botão "Enviar para Assinatura" abre modal com campos: signatário (nome + e-mail, pré-preenchidos do cliente), tipo de documento, mensagem opcional
- [ ] **AC-2:** Ao confirmar, sistema cria envelope no ZapSign via API, gera link de assinatura e registra o documento em `signature_documents` com `status: pending`
- [ ] **AC-3:** Link de assinatura é exibido ao corretor (copiar/enviar via WhatsApp) e enviado automaticamente por e-mail para o signatário via ZapSign
- [ ] **AC-4:** Webhook ZapSign atualiza `status` automaticamente: `pending → signed | refused | expired`
- [ ] **AC-5:** Corretor recebe notificação in-app e e-mail quando cliente assina ("✅ [Nome] assinou a proposta [X]")
- [ ] **AC-6:** Documento assinado (PDF final) é armazenado no Supabase Storage e disponível para download pelo corretor
- [ ] **AC-7:** Status de assinatura visível na listagem de propostas (badge colorido: pendente/assinado/recusado/expirado)
- [ ] **AC-8:** Feature bloqueada para plano Solo via `<PlanGate>` com copy: "Assinatura digital disponível no plano Profissional"
- [ ] **AC-9:** `ZAPSIGN_API_TOKEN` configurável via variável de ambiente; endpoint `/api/webhooks/zapsign` valida token de origem

## Technical Spec

### Database

```sql
CREATE TABLE signature_documents (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  broker_id UUID NOT NULL REFERENCES brokers(id) ON DELETE CASCADE,
  proposal_id UUID REFERENCES proposals(id) ON DELETE SET NULL,
  zapsign_token TEXT NOT NULL,
  zapsign_doc_uuid TEXT NOT NULL UNIQUE,
  signer_name TEXT NOT NULL,
  signer_email TEXT NOT NULL,
  sign_url TEXT NOT NULL,
  status TEXT NOT NULL DEFAULT 'pending'
    CHECK (status IN ('pending', 'signed', 'refused', 'expired')),
  signed_at TIMESTAMPTZ,
  storage_path TEXT,
  created_at TIMESTAMPTZ DEFAULT now(),
  updated_at TIMESTAMPTZ DEFAULT now()
);

CREATE INDEX ON signature_documents (broker_id, status);
CREATE INDEX ON signature_documents (proposal_id);
CREATE INDEX ON signature_documents (zapsign_doc_uuid);

ALTER TABLE signature_documents ENABLE ROW LEVEL SECURITY;
CREATE POLICY "broker own signatures" ON signature_documents
  FOR ALL USING (broker_id = (SELECT id FROM brokers WHERE user_id = auth.uid()));
```

### ZapSign API — Endpoints Utilizados

```typescript
// Criar documento para assinatura
POST https://api.zapsign.com.br/api/v1/docs/
Authorization: Bearer {ZAPSIGN_API_TOKEN}
{
  "name": "Proposta {numero}",
  "signers": [{ "name": "...", "email": "..." }],
  "base64_pdf": "...",  // PDF da proposta convertido para base64
  "lang": "pt-br"
}
// Response: { "open_id", "token", "signers": [{ "token", "sign_url" }] }

// Webhook (recebido no /api/webhooks/zapsign)
// POST com body: { "doc": { "open_id", "status" }, "signer": { ... } }
```

### Server Actions

- `sendForSignature(proposalId, signerName, signerEmail, message?)` → cria envelope ZapSign, insere em `signature_documents`
- `getSignatureDocuments(proposalId)` → lista documentos de uma proposta
- `downloadSignedDocument(documentId)` → gera URL temporária do Supabase Storage

### API Routes

- `POST /api/webhooks/zapsign` — recebe eventos ZapSign, valida token, atualiza status, dispara notificação in-app
- `GET /api/pdf/proposal/[id]` — gera PDF da proposta em base64 para envio ao ZapSign

### UI Components

- `SendForSignatureModal` — modal com seleção de signatário + confirmação
- Badge de status em `ProposalCard` e detalhe da proposta
- Seção "Documentos para Assinatura" no detalhe da proposta
- Download button para PDF assinado

---

## Scope

**IN:**
- Assinatura de propostas aprovadas
- 1 signatário por documento (cliente)
- Notificação in-app + e-mail ao corretor ao assinar
- Download do PDF assinado
- Webhook de status automático

**OUT:**
- Múltiplos signatários por documento (fase futura)
- Assinatura de apólices avulsas (apenas propostas nesta story)
- Geração de apólice pós-assinatura (story separada)
- Upload de documentos externos para assinatura

## Risks

- ZapSign API pode ter latência variável — usar loading state no modal e verificação por webhook (não polling)
- PDF de proposta deve ter tamanho razoável para base64 — limitar a 5 MB
- Webhook sem autenticação forte — validar via `zapsign_doc_uuid` + secret header

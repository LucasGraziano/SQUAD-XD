## User Story

Como corretor que acabou de cadastrar suas apólices, quero ser notificado de forma especial quando meu primeiro alerta de renovação é disparado — para entender que o sistema está trabalhando por mim automaticamente.

---

## Value Proposition

- O "first win" é o momento em que o usuário percebe o valor real do produto pela primeira vez
- Corretores cadastram apólices e ficam sem feedback; não sabem se o sistema "está funcionando"
- Uma notificação de celebração no primeiro alerta = "aha moment" = redução de churn em 30 dias
- Referência: Duolingo streak, Notion first page, Linear first issue — todos têm "first win" celebrations
- Custo de implementação: baixo; impacto em ativação/retenção: alto

---

## Acceptance Criteria

- [ ] **AC-1:** Tabela `brokers` recebe coluna `first_alert_fired_at TIMESTAMPTZ` (null = nunca disparou)
- [ ] **AC-2:** Quando o job de alertas dispara pela primeira vez para um broker (qualquer alerta), registrar `first_alert_fired_at = now()`
- [ ] **AC-3:** Após a gravação de `first_alert_fired_at`, criar uma notificação in-app especial: tipo `first_win`, não-descartável por 48h
- [ ] **AC-4:** Toast/banner de celebração na próxima sessão do corretor: "Seu primeiro alerta foi disparado! O sistema está trabalhando por você 24h."
- [ ] **AC-5:** Notificação inclui: ícone de troféu, nome do cliente cujo alerta foi disparado, CTA "Ver alertas"
- [ ] **AC-6:** A celebração aparece apenas UMA VEZ por broker (idempotente — baseado em `first_alert_fired_at`)
- [ ] **AC-7:** Brokers que já têm alertas disparados (dados históricos) não recebem retroativamente — `first_alert_fired_at` fica null até o próximo disparo
- [ ] **AC-8:** Componente `FirstWinBanner.tsx` é renderizado no layout do dashboard (após `/app/layout.tsx`) apenas quando `first_alert_fired_at` foi definido nas últimas 48h e o banner ainda não foi visto (`first_win_seen_at` null)

### Hook no Job de Alertas

```typescript
// src/lib/alerts/renewal-alerts.ts (atualizar função que dispara alertas)
// Após INSERT bem-sucedido de um alerta:

const broker = await supabase
  .from('brokers')
  .select('first_alert_fired_at')
  .eq('id', brokerId)
  .single()

if (!broker.data?.first_alert_fired_at) {
  await supabase.from('brokers').update({
    first_alert_fired_at: new Date().toISOString()
  }).eq('id', brokerId)

  await supabase.from('notifications').insert({
    broker_id: brokerId,
    type: 'first_win',
    title: 'Primeiro alerta disparado!',
    body: `O sistema está trabalhando por você. Cliente: ${clientName} — vence em ${daysUntilExpiry} dias.`,
    action_url: '/alertas',
    expires_at: new Date(Date.now() + 48 * 60 * 60 * 1000).toISOString()
  })
}
```

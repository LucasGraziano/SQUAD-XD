## User Story

Como corretor novo no Premia, quero ser guiado pelos primeiros passos essenciais com uma checklist visual clara, para chegar ao meu primeiro "aha moment" (ver alertas da minha carteira) em menos de 15 minutos sem precisar de ajuda.

---

## Value Proposition

- PRD O4/KR1: "tempo até aha moment < 15 min" — impossível sem onboarding guiado
- Corretores que chegam ao aha moment nos primeiros 15 min têm retenção 3x maior (benchmark SaaS B2B)
- Sem checklist, o novo usuário abre o dashboard vazio e fecha o browser
- Gamificação (progress bar + celebração) aumenta conclusão de onboarding em 40–60% vs. simples lista
- Desaparece automaticamente após concluído — não polui a UI de usuários veteranos

---

## Acceptance Criteria

- [ ] **AC-1:** Ao primeiro login (broker sem apólices), banner de onboarding aparece no topo do dashboard com progress bar (X/5 passos)
- [ ] **AC-2:** 5 passos em ordem: (1) Preencher perfil + SUSEP, (2) Cadastrar primeiro cliente, (3) Cadastrar primeira apólice OU importar CSV, (4) Ver central de alertas, (5) Compartilhar portal com cliente
- [ ] **AC-3:** Cada passo tem: ícone de status (✅ done / ○ pendente), título curto, CTA com link direto para a tela correta
- [ ] **AC-4:** Passo marcado como concluído automaticamente quando a ação é realizada (sem botão manual)
- [ ] **AC-5:** Ao completar todos os 5 passos → animação de celebração (confetti leve) + badge "Setup completo" no perfil
- [ ] **AC-6:** Checklist salva estado no banco — não reinicia ao recarregar a página
- [ ] **AC-7:** Botão "Fechar" disponível a qualquer momento — estado `dismissed: true` no banco, não aparece mais
- [ ] **AC-8:** Se broker já tem ≥ 1 apólice no cadastro (importado antes do onboarding), passo 3 já começa marcado
- [ ] **AC-9:** Progress bar mostra % completo; cor muda de cinza → verde à medida que passos são completados
- [ ] **AC-10:** Mobile-first — checklist legível e funcional em viewport de 375px

## Technical Spec

### DB — coluna `onboarding_progress` em `brokers`

```sql
-- supabase/migrations/20260514000001_onboarding_progress.sql
ALTER TABLE brokers
  ADD COLUMN onboarding_progress JSONB DEFAULT '{
    "profile": false,
    "first_client": false,
    "first_apolice": false,
    "viewed_alertas": false,
    "shared_portal": false,
    "dismissed": false,
    "completed_at": null
  }';
```

### Server Actions

```typescript
// src/app/actions/onboarding.ts
export async function updateOnboardingStep(step: OnboardingStep, value: boolean)
export async function dismissOnboarding()
```

### Trigger automático por evento

| Evento | Step marcado |
|--------|-------------|
| `configuracoes` salvo com SUSEP preenchido | `profile: true` |
| Primeiro cliente inserido em `clients` | `first_client: true` |
| Primeira apólice inserida em `apolices` | `first_apolice: true` |
| Primeira visita a `/alertas` | `viewed_alertas: true` |
| Primeiro portal gerado/copiado | `shared_portal: true` |

### Componentes

```
src/components/onboarding/
  OnboardingChecklist.tsx   — Server Component (lê progress do broker)
  OnboardingStep.tsx        — Client Component (step individual + CTA)
  OnboardingProgress.tsx    — barra de progresso animada
  OnboardingCelebration.tsx — confetti ao completar (use canvas-confetti)
```

### Integração no Dashboard

```tsx
// src/app/(dashboard)/dashboard/page.tsx
// Adicionar acima do grid principal:
<OnboardingChecklist brokerId={broker.id} progress={broker.onboarding_progress} />
```

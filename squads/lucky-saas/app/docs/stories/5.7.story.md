## User Story

Como corretor, quero enviar mensagens automáticas pelo WhatsApp para meus clientes sem precisar configurar Meta Business API, para ter alertas de renovação e mensagens manuais funcionando em minutos — sem burocracia de aprovação de templates.

---

## Por que Z-API (não Meta Cloud API)

| Critério | Z-API | Meta Cloud API |
|---------|-------|----------------|
| Setup | QR Code (5 min) | WABA + Business verification (semanas) |
| Mensagens livres | Sim | Não (só templates aprovados) |
| Custo | ~R$60–150/mês plano fixo | Por mensagem (~R$0,10 marketing) |
| Aprovação Meta | Não necessária | Necessária |
| Ideal para | Corretores pequenos/médios | Empresas com 10k+ msgs/mês |
| Risco | Conta pode ser banida se spam | Nenhum (oficial Meta) |

> **Decisão:** Z-API para MVP e plano Equipe. Meta API pode ser adicionada como opção premium futura.
> O corretor usa o próprio número de WhatsApp — a Premia apenas gerencia a conexão via Z-API.

---

## Value Proposition

- WhatsApp tem 95% de abertura vs ~20% de email
- Renovações perdidas por "não vi o email" custam clientes reais
- Setup de 5 minutos via QR Code — sem burocracia
- Feature exclusiva do plano Equipe (R$197) — anchor de valor máximo do plano

## Technical Spec

### Z-API

Z-API é um serviço brasileiro que cria uma instância do WhatsApp Web acessível via REST API.

**Base URL:** `https://api.z-api.io/instances/{instanceId}/token/{token}/`

**Endpoints usados:**

```
GET  /qr-code/image   → retorna QR code para escanear
GET  /status          → verifica se está conectado
POST /send-text       → envia mensagem de texto
GET  /disconnect      → desconecta a instância
POST /webhooks        → configura webhook para status de entrega
```

**Exemplo — enviar mensagem:**
```json
POST /send-text
{
  "phone": "5511999999999",
  "message": "Olá João, sua apólice de Auto vence em 30 dias. Precisa renovar?"
}
```

**Resposta:**
```json
{ "zaapId": "3999984263738880DE", "messageId": "3EB09...", "id": "true" }
```

### Database

```sql
CREATE TABLE whatsapp_settings (
  broker_id UUID PRIMARY KEY REFERENCES brokers(id),
  zapi_instance_id TEXT NOT NULL,
  zapi_token TEXT NOT NULL,
  zapi_client_token TEXT NOT NULL,   -- security token do header Client-Token
  is_connected BOOLEAN DEFAULT false,
  connected_phone TEXT,              -- número conectado (preenchido após QR scan)
  webhook_url TEXT,
  created_at TIMESTAMPTZ DEFAULT now(),
  updated_at TIMESTAMPTZ DEFAULT now()
);

CREATE TABLE whatsapp_logs (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  broker_id UUID NOT NULL REFERENCES brokers(id),
  recipient_phone TEXT NOT NULL,
  recipient_name TEXT,
  message_text TEXT NOT NULL,
  trigger_type TEXT NOT NULL,    -- 'manual' | 'renewal_30d' | 'birthday'
  reference_id UUID,             -- client_id ou policy_id relacionado
  zapi_message_id TEXT,
  status TEXT DEFAULT 'sent',    -- 'sent' | 'delivered' | 'read' | 'failed'
  sent_at TIMESTAMPTZ DEFAULT now()
);
```

### Routes

```
GET  /api/webhooks/zapi            → recebe status updates (delivery/read)
GET  /api/zapi/qr                  → proxy para buscar QR code (protege token)
GET  /api/zapi/status              → verifica status de conexão
```

### Environment Variables Required

```
ZAPI_DEFAULT_CLIENT_TOKEN=   # Client-Token header (segurança Z-API)
```

> Os `instance_id` e `token` são por corretor, salvos em `whatsapp_settings`.
> A Premia precisa de 1 conta Z-API master para gerenciar múltiplas instâncias,
> ou cada corretor cria sua própria conta Z-API (modelo self-serve preferível para escalar).

### Modelo Self-Serve (recomendado para MVP)

O corretor cria sua própria conta em z-api.io, cria uma instância, e fornece no Premia:
- Instance ID
- Token

Isso elimina custo de infraestrutura da Premia e responsabiliza o corretor pelo plano Z-API.

## Notes

- Z-API cobra por instância ativa. No modelo self-serve, custo fica com o corretor.
- Número conectado precisa ter WhatsApp Business ou pessoal ativo no celular.
- Se o celular ficar sem internet por >10 min, a instância pode desconectar — avisar no UX.
- Para futuro: opção de migrar para Meta Cloud API (oficial) como opção avançada para corretoras maiores.

## Contexto

O maior valor do Deal Room não é a UI bonita — é o sistema que trabalha **por conta própria**. Esta story implementa os side effects automáticos que disparam quando um deal muda de estágio, eliminando o trabalho manual que hoje o corretor faz (ou esquece de fazer).

---

## Acceptance Criteria

- [x] **AC-1:** Deal vai para `sent` → alerta criado automaticamente: "Seguimento — [Cliente] ainda não aprovou" agendado para +5 dias úteis (se não avançar para `approved` antes)
- [x] **AC-2:** Deal vai para `submitted` com `response_deadline` preenchido → alerta criado: "Seguradora não respondeu" agendado para `response_deadline + 1 dia`
- [x] **AC-3:** Deal vai para `submitted` sem `response_deadline` → alerta criado para +10 dias como fallback
- [x] **AC-4:** Deal vai para `contracted` → alertas de seguimento pendentes do deal são cancelados automaticamente
- [x] **AC-5:** Deal vai para `rejected` → alertas pendentes cancelados; deal some do Kanban imediatamente
- [x] **AC-6:** Quando `view_count` passa de 0 para 1 (cliente visualizou o link pela primeira vez) → notificação in-app criada: "Cliente [Nome] visualizou sua proposta agora"
- [x] **AC-7:** Deal fica em qualquer stage sem mudança por 30 dias → alerta de revisão: "Deal [Cliente/Ramo] está parado há 30 dias"
- [x] **AC-8:** Todos os triggers são implementados dentro de `updateDealStage` (server action) — sem cron job separado para os triggers síncronos
- [x] **AC-9:** Alertas gerados usam a tabela `alerts` existente com `type = 'deal_followup'`
- [x] **AC-10:** Alertas de deal parado (AC-7) via cron job existente ou novo endpoint `/api/cron/deal-stale-check`

## Technical Spec

### Triggers em updateDealStage

```typescript
// Dentro de updateDealStage, após atualizar o status:

if (newStage === 'sent') {
  // Criar alerta de seguimento em +5 dias úteis
  const followupDate = addBusinessDays(new Date(), 5)
  await sb.from('alerts').insert({
    broker_id: brokerId,
    deal_id: dealId,
    type: 'deal_followup',
    title: `Seguimento — ${deal.clients?.name} ainda não aprovou`,
    scheduled_for: followupDate.toISOString().split('T')[0],
    status: 'pending',
    metadata: { stage_trigger: 'sent', cancel_on_stage: 'approved' }
  })
}

if (newStage === 'submitted') {
  const deadline = deal.response_deadline
    ? new Date(deal.response_deadline)
    : addDays(new Date(), 10)
  deadline.setDate(deadline.getDate() + 1)
  await sb.from('alerts').insert({
    broker_id: brokerId,
    deal_id: dealId,
    type: 'deal_insurer_followup',
    title: `Seguradora sem resposta — ${deal.clients?.name}`,
    scheduled_for: deadline.toISOString().split('T')[0],
    status: 'pending',
    metadata: { stage_trigger: 'submitted', cancel_on_stage: 'issued' }
  })
}

if (newStage === 'contracted' || newStage === 'rejected') {
  // Cancelar alertas pendentes deste deal
  await sb.from('alerts')
    .update({ status: 'canceled' })
    .eq('deal_id', dealId)
    .eq('status', 'pending')
}
```

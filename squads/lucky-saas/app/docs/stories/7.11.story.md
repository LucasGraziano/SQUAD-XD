## User Story

Como corretor, quero ver um score de saúde da minha carteira — tipo "sua carteira está 72% atualizada" — para entender rapidamente quais áreas precisam de atenção e me sentir motivado a melhorar.

---

## Value Proposition

- Corretores com muitas apólices perdem a visão global: "Estou bem? Estou atrasado?"
- Um score único transforma dados espalhados em uma métrica acionável
- Gamificação: o corretor quer aumentar o score de 72% para 100% — gera DAU
- Score baixo = urgência = corretor abre alertas, age nas renovações
- Referencial no produto: LinkedIn SSI score, Duolingo streak, Notion usage %
- Sem nova tabela — calculado em tempo real a partir de dados existentes

---

## Acceptance Criteria

- [ ] **AC-1:** Widget "Saúde da Carteira" no dashboard principal, exibindo score de 0-100%
- [ ] **AC-2:** Score calculado com 4 componentes com pesos iguais (25% cada):
  - **Cobertura de alertas (25%):** % de apólices com alerta de renovação configurado
  - **Atualização de dados (25%):** % de apólices com vigência_fim > hoje (ativas, não expiradas)
  - **Clientes completos (25%):** % de clientes com e-mail E telefone preenchidos
  - **Engajamento recente (25%):** se o broker fez login nos últimos 14 dias (0 ou 25%)
- [ ] **AC-3:** Score exibido como barra de progresso colorida: vermelho (0-49%), amarelo (50-74%), verde (75-100%)
- [ ] **AC-4:** Abaixo do score, 3 "dicas de melhoria" priorizadas pelo componente com menor sub-score: "X apólices sem alerta configurado → Resolver"
- [ ] **AC-5:** Cada dica tem link direto para a ação: `→ Ver apólices sem alerta`, `→ Completar clientes`, etc.
- [ ] **AC-6:** Score é recalculado a cada carregamento do dashboard (não cacheado por mais de 5 min)
- [ ] **AC-7:** Widget exibe data de última atualização: "Atualizado agora"
- [ ] **AC-8:** Tooltip explicando cada componente ao hover no score

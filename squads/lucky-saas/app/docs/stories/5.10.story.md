## User Story

Como corretor, quero cotar automaticamente nas 10 principais seguradoras do mercado diretamente dentro do Premia, para gerar um comparativo completo em segundos sem precisar acessar portal por portal manualmente.

---

## Value Proposition

- Corretor perde 2–4h/dia acessando portais individuais para cotar
- Feature que sozinha justifica a assinatura Equipe (R$197/mês)
- Comparativo profissional em PDF já integrado (Story anterior)
- Posicionamento: único SaaS de corretora com multicálculo real no Brasil nesta faixa de preço
- Upsell natural: corretores Solo migram para Equipe quando veem a demo

## Acceptance Criteria

- [ ] **AC-1:** Corretor pode cadastrar suas credenciais de portal para cada seguradora em `/configuracoes/seguradoras` (campos: seguradora, login, senha — criptografados em repouso com `pgp_sym_encrypt`)
- [ ] **AC-2:** Na tela de Cotações, corretor preenche o formulário de cotação unificado (campos: ramo, veículo/objeto, dados do cliente, cobertura desejada) e seleciona quais seguradoras cotar
- [ ] **AC-3:** Sistema dispara cotações em paralelo para todas as seguradoras selecionadas via Playwright (headless) ou API REST — timeout por seguradora: 30s
- [ ] **AC-4:** Resultados aparecem progressivamente em tempo real conforme chegam (Server-Sent Events ou polling a cada 2s)
- [ ] **AC-5:** Cada resultado mostra: seguradora, prêmio total, franquia, coberturas incluídas, vigência, status (aguardando / concluído / falhou)
- [ ] **AC-6:** Corretor pode salvar qualquer cotação como proposta (`quotes` table) com 1 clique
- [ ] **AC-7:** Da cotação salva, gera PDF comparativo (QuoteComparisonDocument — já implementado)
- [ ] **AC-8:** Cotações ficam salvas por 7 dias em histórico em `/cotacoes`
- [ ] **AC-9:** Se credencial de uma seguradora falhar, o sistema marca como "Falhou — verificar credenciais" sem travar as demais
- [ ] **AC-10:** Funcionalidade exclusiva plano Equipe — upgrade wall para Solo e Profissional

## Technical Spec

### Arquitetura

```
Browser → POST /api/cotacao/iniciar → cria job no DB → retorna jobId
Browser → GET /api/cotacao/[jobId]/stream (SSE) → recebe resultados progressivos
Server → Playwright workers paralelos (1 por seguradora) → atualiza DB → SSE notifica
```

### Stack Técnica

- **Playwright** (já disponível via MCP) para portais sem API
- **Fetch/Axios** para seguradoras com API REST
- **SSE (Server-Sent Events)** via Next.js Route Handler para streaming de resultados
- **Supabase Realtime** como alternativa ao SSE se limitações de edge function
- **pgcrypto** para criptografar credenciais dos portais

### Database

```sql
-- Credenciais dos portais (por corretor, por seguradora)
CREATE TABLE insurer_credentials (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  broker_id UUID NOT NULL REFERENCES brokers(id),
  seguradora TEXT NOT NULL,            -- 'porto_seguro', 'sulamerica', etc.
  portal_login TEXT NOT NULL,
  portal_password_enc TEXT NOT NULL,   -- pgp_sym_encrypt(password, app_secret)
  is_active BOOLEAN DEFAULT true,
  last_tested_at TIMESTAMPTZ,
  last_test_status TEXT,               -- 'ok' | 'falhou'
  created_at TIMESTAMPTZ DEFAULT now(),
  UNIQUE(broker_id, seguradora)
);

-- Jobs de cotação
CREATE TABLE quote_jobs (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  broker_id UUID NOT NULL REFERENCES brokers(id),
  ramo TEXT NOT NULL,
  input_data JSONB NOT NULL,           -- dados do formulário de cotação
  seguradoras TEXT[] NOT NULL,         -- quais seguradoras foram solicitadas
  status TEXT DEFAULT 'running',       -- 'running' | 'completed' | 'partial'
  created_at TIMESTAMPTZ DEFAULT now(),
  completed_at TIMESTAMPTZ,
  expires_at TIMESTAMPTZ DEFAULT now() + INTERVAL '7 days'
);

-- Resultados individuais por seguradora
CREATE TABLE quote_results (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  job_id UUID NOT NULL REFERENCES quote_jobs(id),
  broker_id UUID NOT NULL REFERENCES brokers(id),
  seguradora TEXT NOT NULL,
  status TEXT DEFAULT 'pending',       -- 'pending' | 'ok' | 'falhou' | 'timeout'
  premio_total NUMERIC,
  franquia NUMERIC,
  coberturas TEXT,                     -- CSV de coberturas incluídas
  vigencia_meses INT,
  raw_data JSONB,                      -- dados brutos da resposta
  error_message TEXT,
  fetched_at TIMESTAMPTZ
);
```

### Formulário Unificado de Cotação

Campos variam por ramo:

**Auto:**
- Placa ou ano/modelo/fabricante
- CEP do segurado
- Tipo de uso (particular/comercial)
- Perfil condutor (idade, sexo, estado civil)
- Coberturas desejadas (compreensiva/colisão/terceiros)

**Residencial:**
- CEP, tipo (casa/apto), m² construído
- Valor do imóvel
- Coberturas desejadas

**Vida:**
- Idade do segurado, sexo
- Capital segurado desejado
- Tipo (individual/empresarial)

### Routes

```
POST /api/cotacao/iniciar          → cria quote_job, dispara workers
GET  /api/cotacao/[jobId]/stream   → SSE com resultados progressivos
GET  /api/cotacao/[jobId]          → snapshot atual do job
```

## Notes

- **Scraping autenticado**: o corretor usa suas próprias credenciais de portal. A Premia apenas automatiza o que o corretor faria manualmente. Responsabilidade do corretor aceitar os termos dos portais.
- **Detecção de bot**: alguns portais têm proteção. Playwright com `stealth` (randomização de UA, delays humanos) mitiga.
- **Custos**: Playwright rodando em servidor pode ser caro. Considerar worker pool limitado e queue para escalabilidade.
- **Prioridade de ramo Auto**: 80% das cotações de corretoras independentes são de auto seguro.
- **Sem selenium/webdriver**: usar Playwright (mais moderno, menos detectável).

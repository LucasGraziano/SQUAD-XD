# Story 7.10 — Empty States Inteligentes com CTA Guiado

**Status:** Ready for Review
**Epic:** 7 — Core Loop & Retenção
**Sprint:** 7
**Agent:** @dev (Dex)
**Prioridade:** P1 — Reduz abandono; guia o usuário até o primeiro valor em cada módulo
**Created:** 2026-05-14

---

## User Story

Como corretor novo no Premia, quando abro um módulo vazio (clientes, apólices, cotações, pipeline), quero ver uma tela que me explica o que fazer em seguida — não uma lista em branco — para não ficar perdido e começar a usar o sistema mais rápido.

---

## Value Proposition

- "Empty list" é o maior ponto de abandono em SaaS B2B: o usuário não sabe o que fazer
- Empty states com CTA contextual reduzem time-to-first-value em média 40% (benchmark Intercom)
- Cada módulo tem um empty state diferente adaptado ao contexto (cliente, apólice, alerta, etc.)
- Hoje o Premia mostra listas vazias sem orientação — o usuário fica perdido
- Custo: baixo (UI only, sem DB); retorno: alto (ativação + onboarding)

---

## Acceptance Criteria

- [ ] **AC-1:** Módulo `/clientes` vazio exibe: ilustração + "Cadastre seu primeiro cliente para começar" + botão "Novo cliente"
- [ ] **AC-2:** Módulo `/apolices` vazio exibe: "Importe suas apólices ou cadastre a primeira manualmente" + 2 CTAs: "Importar CSV" (Story 7.2) e "Nova apólice"
- [ ] **AC-3:** Módulo `/alertas` vazio exibe: "Nenhum alerta ainda — o sistema monitora seus vencimentos automaticamente. Adicione apólices para ativar." + CTA "Ver apólices"
- [ ] **AC-4:** Módulo `/cotacoes` vazio exibe: "Crie sua primeira proposta e envie ao cliente em minutos" + CTA "Nova cotação"
- [ ] **AC-5:** Módulo `/pipeline` vazio exibe: "Seu pipeline de vendas começa aqui. Adicione leads e acompanhe cada negociação." + CTA "Novo lead"
- [ ] **AC-6:** Empty states usam ilustração SVG coerente com o design system (paleta e estilo do Premia)
- [ ] **AC-7:** Em telas de busca/filtro sem resultado (lista filtrada vazia), exibir empty state diferente: "Nenhum resultado para [termo]. [Limpar filtros]" — sem ilustração, mensagem inline
- [ ] **AC-8:** Empty states são responsivos e mobile-first

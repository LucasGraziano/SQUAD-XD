## Contexto

Duas entregas em uma story de tamanho S cada:

1. **DealTimeline** — componente visual que mostra o histórico de estágios de um deal, consumindo `deal_stage_history`. Usado dentro do Deal Workspace (8.4).

2. **Deprecar /propostas** — a página `/propostas` (Story 5.13) rastreava proposta à seguradora separadamente. Com o Deal Room, esses campos migraram para `deals` (stages `submitted`/`under_analysis`/`issued`). A página recebe um redirect e um banner de migração.

---

## Acceptance Criteria

### DealTimeline

- [x] **AC-1:** Componente `DealTimeline` recebe `dealId` e carrega `deal_stage_history` via `getDealTimeline(dealId)`
- [x] **AC-2:** Timeline vertical com marcador de ponto por estágio, ordenada cronologicamente (mais antigo no topo)
- [x] **AC-3:** Cada entrada mostra: ícone do stage (cor do `DEAL_STAGE_COLORS`), label do stage (`DEAL_STAGE_LABELS`), data/hora formatada em pt-BR, nota (se presente)
- [x] **AC-4:** Entrada de criação do deal sempre aparece como primeiro item ("Deal criado")
- [x] **AC-5:** Stage atual destacado com indicador "Agora" e animação de pulse
- [x] **AC-6:** Se `view_count > 0`, inserir entrada especial: "Cliente visualizou pela primeira vez" com `last_viewed_at`
- [x] **AC-7:** Timeline colapsável em mobile (padrão fechado, expande ao clicar)

### Deprecar /propostas

- [x] **AC-8:** `/propostas` exibe banner no topo: "Esta página foi integrada ao Pipeline de Negociações. Suas propostas agora são gerenciadas como Deals." com botão "Ir para o Pipeline"
- [x] **AC-9:** Lista de `proposals` existentes ainda é exibida temporariamente abaixo do banner, com link para o deal correspondente quando `policy_id` ou `lead_id` tiver match em `quote_requests`
- [x] **AC-10:** Link na sidebar para `/propostas` removido — substituído por link para `/pipeline`
- [x] **AC-11:** `/propostas/new` (se existir) redireciona para `/pipeline`

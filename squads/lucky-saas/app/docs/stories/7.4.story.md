## User Story

Como corretor com uma proposta aprovada pelo cliente, quero convertê-la em apólice com 1 clique — com todos os dados já preenchidos — para não precisar redigitar as mesmas informações duas vezes e evitar erros de transcrição.

---

## Value Proposition

- PRD FR-04: "Conversão de proposta aprovada em apólice (1 clique)" — prometido, nunca entregue
- Hoje o corretor fecha a proposta, abre `/apolices`, e recomeça o cadastro do zero — puro retrabalho
- Elimina o principal ponto de fricção pós-venda: re-entrada de dados já existentes
- Fecha o loop comercial completo: Lead (pipeline) → Cotação → Proposta → **Apólice** em um fluxo único
- Reduz erros de transcrição (número de apólice digitado errado, data de vigência trocada)
- Depende de Story 7.3 (status flow) para o botão aparecer apenas em propostas `approved`

---

## Acceptance Criteria

- [ ] **AC-1:** Proposta com status `approved` exibe botão primário "Converter em Apólice" (verde, destaque)
- [ ] **AC-2:** Ao clicar → modal de conversão abre pré-preenchido com dados da proposta:
  - Cliente (vinculado)
  - Ramo
  - Seguradora (primeira marcada como "Recomendada" no multicálculo, ou única se só uma)
  - Prêmio (valor da seguradora recomendada)
  - Vigência início (hoje por default, editável)
  - Vigência fim (calculada: início + 12 meses, editável)
  - Comissão % (field editável, default do ramo se configurado)
- [ ] **AC-3:** Campos pré-preenchidos são editáveis antes de confirmar — corretor pode ajustar o número da apólice (número real da seguradora, não disponível antes da emissão)
- [ ] **AC-4:** Campo obrigatório adicional: "Número da apólice" (emitido pela seguradora — não existe na proposta)
- [ ] **AC-5:** Ao confirmar: apólice criada em `apolices`; proposta atualizada para status `contracted`; `apolice_id` linkado na proposta; alertas de renovação gerados automaticamente
- [ ] **AC-6:** Toast de sucesso: "Apólice #[número] criada com sucesso · Alertas de renovação configurados"
- [ ] **AC-7:** Proposta convertida exibe link "Ver Apólice" na listagem de cotações (usando `apolice_id`)
- [ ] **AC-8:** Apólice criada aparece imediatamente na listagem `/apolices`
- [ ] **AC-9:** Operação reversível apenas por exclusão da apólice (não há "desfazer conversão") — aviso no modal
- [ ] **AC-10:** Bloqueado para plano Solo via `<PlanGate requiredPlan="profissional">` (cotações são Pro+)

## Technical Spec

### Server Action

```typescript
// src/app/actions/quotes.ts (adicionar)
export async function convertQuoteToApolice(
  quoteId: string,
  data: ConvertQuoteData
): Promise<{ apoliceId: string }>

interface ConvertQuoteData {
  numero_apolice: string
  vigencia_inicio: string  // ISO date
  vigencia_fim: string     // ISO date
  comissao_pct?: number
  premio_override?: number // se corretor editar o prêmio
}

// Internamente:
// 1. Validar que quote está em status 'approved' e pertence ao broker
// 2. Buscar quote + item recomendado (is_recommended: true) ou primeiro item
// 3. Buscar client_id da quote
// 4. INSERT em apolices com dados combinados
// 5. UPDATE quote_requests SET status='contracted', apolice_id=newId, contracted_at=now()
// 6. INSERT em quote_status_history
// 7. Gerar alertas de renovação (60, 30, 15, 7, 1 dias)
// 8. Retornar { apoliceId }
```

### Mapeamento de dados Proposta → Apólice

| Campo Proposta | Campo Apólice | Observação |
|---------------|--------------|------------|
| `client_id` | `client_id` | direto |
| `ramo` | `ramo` | direto |
| `quote_items[recommended].seguradora` | `seguradora` | item recomendado |
| `quote_items[recommended].premium_total` | `premio` | editável no modal |
| *(hoje)* | `vigencia_inicio` | default hoje, editável |
| *(hoje + 12m)* | `vigencia_fim` | editável |
| *(input usuário)* | `numero_apolice` | obrigatório, não existe na proposta |

## Scope

**IN:**
- Botão "Converter em Apólice" em propostas approved
- Modal com dados pré-preenchidos e editáveis
- Criação de apólice + atualização de status da proposta
- Geração automática de alertas de renovação
- Link "Ver Apólice" na proposta convertida

**OUT:**
- Editar apólice criada a partir da conversão (usar `/apolices/[id]` existente)
- Conversão em lote (múltiplas propostas de uma vez)
- Desfazer conversão (usar exclusão de apólice)
- Criar pendência automática pós-conversão (futura melhoria)

## Risks

- Se a proposta tem múltiplas seguradoras sem "Recomendada" marcada, usar a primeira — comunicar ao corretor no modal qual foi selecionada
- Vigência padrão de 12 meses pode não ser correta para todos os ramos (vida pode ser anual ou vitalício) — campo editável resolve, mas colocar aviso para ramos especiais

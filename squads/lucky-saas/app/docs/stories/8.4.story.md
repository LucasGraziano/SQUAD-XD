# Story 8.4 — Deal Room: Deal Workspace (/deals/[id])

**Status:** Done

**Sprint:** 8

**Prioridade:** P0 — Substitui /cotacoes e /propostas como centro de trabalho de um deal
**Created:** 2026-05-18
**Depende de:** 8.1, 8.2

---

## Contexto

Hoje o corretor precisa navegar entre `/cotacoes` (multicálculo), `/propostas` (protocolo da seguradora) e `/alertas` (renovação) para trabalhar em uma única negociação. O Deal Workspace centraliza tudo em uma única página: o multicálculo, o status flow, os campos da seguradora, o link de compartilhamento, e o botão de conversão para apólice.

A URL `/cotacoes/[id]` passa a redirecionar para `/deals/[id]`.

---

## Acceptance Criteria

- [x] **AC-1:** Página `/deals/[id]` criada como server component que carrega o deal completo via `getDeal(id)`
- [x] **AC-2:** Header do workspace: nome do cliente + ramo, badge de stage atual (editável via dropdown inline), breadcrumb "Pipeline > [Cliente] > [Ramo]"
- [x] **AC-3:** Seção **Multicálculo** (existente de `/cotacoes`): lista de `deal_items` por seguradora, botões adicionar/remover, marcar como recomendada, gerar PDF comparativo — funcionalidade idêntica ao `CotacoesClient` existente
- [x] **AC-4:** Seção **Compartilhamento com Cliente** (expandida de `/cotacoes`): link do deal (`/q/[share_token]`), botão Copiar, botão WhatsApp, indicador "Cliente visualizou" com timestamp, status de aprovação
- [x] **AC-5:** Seção **Seguradora** — aparece apenas quando stage ≥ `submitted`: campos editáveis `proposal_number`, `protocol_number`, `response_deadline`; badge de prazo (verde se dentro do prazo, vermelho se vencido)
- [x] **AC-6:** Botão de ação primário contextual por stage:
  - `prospecting` / `draft` → "Enviar para Cliente" (muda para `sent`, copia link)
  - `sent` → "Marcar como Aprovada" (muda para `approved`)
  - `approved` → "Enviar à Seguradora" (abre mini-form para `proposal_number` + muda para `submitted`)
  - `submitted` → "Marcar Em Análise" (muda para `under_analysis`)
  - `under_analysis` → "Seguradora Emitiu" (muda para `issued`)
  - `issued` → "Registrar Apólice" (abre `ConvertToApoliceModal`)
- [x] **AC-7:** Botão secundário "Recusar / Arquivar" disponível em todos os stages (exceto `contracted`) — abre modal de confirmação com campo `rejection_reason`
- [x] **AC-8:** Seção **Timeline** ao lado direito (ou aba inferior em mobile) — componente `DealTimeline` da Story 8.5
- [x] **AC-9:** Seção **Apólice Vinculada** — aparece quando `policy_id` existe; exibe número, vigência, link para `/apolices/[id]`
- [x] **AC-10:** `/cotacoes/[id]` redireciona para `/deals/[id]` via `redirect()` no server component
- [x] **AC-11:** `/cotacoes` (listagem) mantida temporariamente como alias de `/deals` — redirect também
- [x] **AC-12:** Mobile-first: seções em accordion colapsável em telas < 768px

## Technical Spec

### Estrutura de layout

```
/deals/[id]
├── Header (breadcrumb + stage badge + ações)
├── [grid 2 colunas em desktop, 1 em mobile]
│   ├── Coluna Principal (2/3)
│   │   ├── Seção Multicálculo
│   │   ├── Seção Compartilhamento
│   │   └── Seção Seguradora (condicional por stage)
│   └── Coluna Lateral (1/3)
│       ├── DealTimeline (Story 8.5)
│       └── Apólice Vinculada (condicional)
└── Botões de ação flutuantes (mobile) ou no Header (desktop)
```

## User Story

Como corretor, quero ser notificado automaticamente nos aniversários dos meus clientes — com uma sugestão de seguro relevante para o momento de vida deles — para usar o aniversário como pretexto de contato e abrir uma conversa de cross-sell de forma natural e não invasiva.

---

## Value Proposition

- Aniversário é o pretexto de contato mais aceito socialmente — ninguém recusa parabéns
- Combinado com uma sugestão relevante ("Já pensou em proteger sua família com seguro de vida?") → taxa de abertura muito maior que cold outreach
- Corretores top performers fazem isso manualmente; o Premia faz automaticamente
- Referência: CRM bancário e corretoras de alta performance fazem isso via planilha — Premia automatiza
- Cross-sell sugerido é contextual: cliente com filho? → Vida. Cliente que completou 18? → Auto. Cliente 60+? → Saúde

---

## Acceptance Criteria

- [ ] **AC-1:** Campo `data_nascimento` adicionado em `clients` (opcional, sem obrigatoriedade retroativa)
- [ ] **AC-2:** Formulário de cadastro/edição de cliente atualizado para incluir campo "Data de Nascimento"
- [ ] **AC-3:** Job diário verifica clientes cujo aniversário é hoje (`date_part('month', data_nascimento) = date_part('month', CURRENT_DATE) AND date_part('day', ...) = date_part('day', CURRENT_DATE)`)
- [ ] **AC-4:** Para cada cliente aniversariante, criar notificação in-app para o broker: "Hoje é aniversário de [Nome]! 🎂 [Sugestão de cross-sell contextual]"
- [ ] **AC-5:** Sugestão de cross-sell contextual baseada na idade atual do cliente:
  - 18-25 anos → "Aproveite para oferecer Seguro Auto — primeira habilitação"
  - 25-40 anos → "Momento ideal para Seguro de Vida — crescimento de família"
  - 40-60 anos → "Considere ofertar Seguro de Saúde premium"
  - 60+ anos → "Seguro de Vida vitalício ou previdência privada"
  - Qualquer idade → fallback genérico: "Aproveite para entrar em contato e reforçar o relacionamento"
- [ ] **AC-6:** Notificação inclui botão "Enviar parabéns no WhatsApp" → link WhatsApp pré-formatado: "Olá [Nome], feliz aniversário! 🎉 [mensagem personalizada com sugestão]"
- [ ] **AC-7:** Apenas 1 notificação por cliente por ano (não repetir no mesmo aniversário se já enviado)
- [ ] **AC-8:** Clientes sem `data_nascimento` preenchida não geram notificação (silencioso)
- [ ] **AC-9:** Na listagem de clientes, coluna opcional "Próximo aniversário" + filtro "Aniversariantes do mês"

## Technical Spec

### DB — Migration

```sql

ALTER TABLE clients
  ADD COLUMN data_nascimento DATE;

-- Índice para o job diário de aniversários
CREATE INDEX idx_clients_birthday_month_day
  ON clients (
    EXTRACT(MONTH FROM data_nascimento),
    EXTRACT(DAY FROM data_nascimento)
  )
  WHERE data_nascimento IS NOT NULL;

-- Tracking de notificações enviadas (evitar duplicata anual)
CREATE TABLE birthday_notifications_log (
  id UUID DEFAULT gen_random_uuid() PRIMARY KEY,
  client_id UUID NOT NULL REFERENCES clients(id) ON DELETE CASCADE,
  broker_id UUID NOT NULL REFERENCES brokers(id) ON DELETE CASCADE,
  year INT NOT NULL,  -- ano do aniversário notificado
  notified_at TIMESTAMPTZ DEFAULT now(),
  UNIQUE(client_id, year)  -- 1 por cliente por ano
);
```

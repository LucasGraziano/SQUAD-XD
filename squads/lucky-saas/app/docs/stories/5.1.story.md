## User Story

Como corretor, quero gerar um link personalizado para cada cliente, onde ele pode visualizar suas próprias apólices ativas, datas de vencimento e documentos — sem precisar ligar para mim.

---

## Value Proposition

- Diferencial vs. concorrentes (Facilita Corretor, Maiseg): nenhum oferece portal de autoatendimento
- Reduz chamadas de "qual o vencimento da minha apólice?"
- Aumenta percepção de profissionalismo do corretor
- Feature exclusiva do plano Profissional (R$97) — driver de upgrade

---

## Acceptance Criteria

- [ ] **AC-1:** Corretor pode gerar token de acesso único por cliente na página de detalhe do cliente
- [ ] **AC-2:** Link público formato: `/portal/[clientToken]` — não requer login
- [ ] **AC-3:** Portal exibe: nome do cliente, apólices ativas (ramo, seguradora, vigência, prêmio), alertas de vencimento próximo
- [ ] **AC-4:** Portal é read-only — cliente não edita nada
- [ ] **AC-5:** Token tem expiração configurável (30/60/90 dias) ou vitalício
- [ ] **AC-6:** Corretor pode revogar acesso do cliente a qualquer momento
- [ ] **AC-7:** Portal exibe logo/nome da corretora (white-label básico)
- [ ] **AC-8:** Portal é mobile-first e responsivo
- [ ] **AC-9:** URL de acesso pode ser enviada diretamente via WhatsApp (botão "Copiar link" + "Enviar via WhatsApp")
- [ ] **AC-10:** Funcionalidade bloqueada no plano Solo (R$47) — exibe upgrade wall

## Technical Spec

### Database

```sql
-- Nova tabela
CREATE TABLE client_portal_tokens (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  broker_id UUID NOT NULL REFERENCES brokers(id),
  client_id UUID NOT NULL REFERENCES clients(id),
  token TEXT NOT NULL UNIQUE DEFAULT encode(gen_random_bytes(32), 'hex'),
  expires_at TIMESTAMPTZ,
  revoked_at TIMESTAMPTZ,
  created_at TIMESTAMPTZ DEFAULT now()
);

-- RLS: público pode ler se token válido (não revogado, não expirado)
ALTER TABLE client_portal_tokens ENABLE ROW LEVEL SECURITY;
```

### Routes

- `app/(portal)/portal/[token]/page.tsx` — Server Component, valida token, carrega dados
- `app/(portal)/portal/[token]/not-found.tsx` — Token inválido/expirado
- `app/(dashboard)/clientes/[id]/page.tsx` — Adicionar seção "Portal do Cliente" com geração de token

### Stack

- Supabase RLS para isolamento de dados
- Next.js route group `(portal)` separado do `(dashboard)` e `(auth)`
- Layout próprio do portal — sem sidebar de corretor

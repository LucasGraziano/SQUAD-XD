## User Story

Como corretor, quero registrar e acompanhar sinistros dos meus clientes, com status e histórico, para não perder o fio de nenhuma ocorrência em andamento — e para o cliente saber que estou cuidando.

---

## Value Proposition

- Gestão de sinistros é processo 100% manual (email/WhatsApp/caderno) para a maioria dos corretores
- Falta de visibilidade sobre sinistros em aberto gera perda de clientes
- Nenhum concorrente direto (Facilita Corretor, Maiseg) tem tracking estruturado
- Feature exclusiva do plano Profissional (R$97) — driver de upgrade

---

## Acceptance Criteria

- [ ] **AC-1:** Corretor pode abrir um sinistro vinculado a uma apólice específica
- [ ] **AC-2:** Campos do sinistro: data de ocorrência, tipo (colisão/roubo/incêndio/etc.), descrição, número do processo na seguradora (opcional), valor estimado (opcional)
- [ ] **AC-3:** Status do sinistro: Aberto → Em Análise → Aguardando Docs → Aprovado → Pago → Encerrado | Negado
- [ ] **AC-4:** Histórico de atualizações de status com timestamp e nota do corretor
- [ ] **AC-5:** Lista de sinistros em aberto visível na página de resumo do cliente
- [ ] **AC-6:** Dashboard mostra contador de sinistros abertos (badge no sidebar ou card no Dashboard)
- [ ] **AC-7:** Corretor pode marcar sinistro como "Aguardando documento" com nota do que falta
- [ ] **AC-8:** Upload de documentos do sinistro (laudos, fotos, BO) via Supabase Storage
- [ ] **AC-9:** Notificação por email ao cliente quando status muda (integração com Story 5.4)
- [ ] **AC-10:** Funcionalidade bloqueada no plano Solo — exibe upgrade wall

## Technical Spec

### Database

```sql
CREATE TABLE claims (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  broker_id UUID NOT NULL REFERENCES brokers(id),
  policy_id UUID NOT NULL REFERENCES policies(id),
  client_id UUID NOT NULL REFERENCES clients(id),
  occurrence_date DATE NOT NULL,
  claim_type TEXT NOT NULL, -- colisao, roubo, incendio, outros
  description TEXT,
  insurer_process_number TEXT,
  estimated_value NUMERIC(12,2),
  status TEXT NOT NULL DEFAULT 'open',
  -- open | analyzing | awaiting_docs | approved | paid | closed | denied
  created_at TIMESTAMPTZ DEFAULT now(),
  updated_at TIMESTAMPTZ DEFAULT now()
);

CREATE TABLE claim_updates (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  claim_id UUID NOT NULL REFERENCES claims(id),
  broker_id UUID NOT NULL REFERENCES brokers(id),
  old_status TEXT,
  new_status TEXT NOT NULL,
  note TEXT,
  created_at TIMESTAMPTZ DEFAULT now()
);

ALTER TABLE claims ENABLE ROW LEVEL SECURITY;
ALTER TABLE claim_updates ENABLE ROW LEVEL SECURITY;
```

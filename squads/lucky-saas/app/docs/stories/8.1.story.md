# Story 8.1 — Deal Room: Schema Evolution (deals + deal_stage_history)

**Status:** Done
**Epic:** 8 — Deal Room: Fluxo Unificado de Negociações
**Sprint:** 8
**Agent:** @dev (Dex)
**Prioridade:** P0 — Foundation de todo o Epic 8
**Created:** 2026-05-18

---

## Contexto

O @analyst (Atlas) mapeou que o fluxo entre Pipeline, Cotações, Propostas e Apólices está fragmentado em 4 páginas que não se comunicam. A raiz do problema é estrutural: temos entidades separadas (`leads`, `quote_requests`, `proposals`) para o que é, na prática, uma única coisa — uma **negociação** (deal).

Esta story faz a evolução do schema para unificar tudo em `deals`, mantendo compatibilidade com dados existentes.

**Decisão de design:** em vez de criar uma tabela nova do zero, expandimos `quote_requests` com os campos que estavam em `proposals` e adicionamos os novos estágios. Os `leads` sem cotação são migrados para `deals` com status `prospecting`. A tabela `proposals` é deprecada.

---

## Acceptance Criteria

- [x] **AC-1:** Coluna `status` em `quote_requests` expandida com novos valores: `prospecting` | `draft` | `sent` | `approved` | `submitted` | `under_analysis` | `issued` | `contracted` | `rejected` — sem quebrar registros existentes (default `draft` mantido)
- [x] **AC-2:** Campos de proposta à seguradora adicionados a `quote_requests`: `proposal_number TEXT`, `protocol_number TEXT`, `response_deadline DATE`, `rejection_reason TEXT`, `submitted_at TIMESTAMPTZ`, `issued_at TIMESTAMPTZ`
- [x] **AC-3:** Tabela `deal_stage_history` criada: `(id, deal_id → quote_requests.id, broker_id, from_stage, to_stage, note, changed_at)` com RLS
- [x] **AC-4:** Coluna `source TEXT` adicionada a `quote_requests` para registrar origem do deal (indicação, site, ligação, etc.)
- [x] **AC-5:** `leads` sem `quote_request` associada migrados para `quote_requests` com `status = 'prospecting'`, preservando `client_id`, `ramo`, `notes`, `created_at`
- [x] **AC-6:** Coluna `lead_id` em `quote_requests` mantida para rastrear origem — leads migrados recebem auto-referência via novo campo `original_lead_id`
- [x] **AC-7:** Tabela `proposals` marcada como deprecated via comentário na migration; dados existentes migrados para os novos campos em `quote_requests` (via `proposal_number`, `protocol_number`, `submitted_at`, etc.)
- [x] **AC-8:** Índices criados: `(broker_id, status)` já existente + novo `(broker_id, submitted_at)` + `(deal_id)` em `deal_stage_history`
- [x] **AC-9:** RLS em `deal_stage_history`: corretor só vê histórico dos próprios deals
- [x] **AC-10:** Todos os campos novos são nullable — zero breaking change em código existente

## Technical Spec

### Migration

```sql
-- supabase/migrations/20260518000010_deal_room_schema.sql

-- 1. Expandir status de quote_requests
ALTER TABLE quote_requests
  DROP CONSTRAINT IF EXISTS quote_requests_status_check;

ALTER TABLE quote_requests
  ADD CONSTRAINT quote_requests_status_check
  CHECK (status IN (
    'prospecting',   -- lead puro, sem cotação montada
    'draft',         -- cotação sendo montada
    'sent',          -- enviada ao cliente
    'approved',      -- cliente aprovou
    'submitted',     -- enviada à seguradora
    'under_analysis',-- seguradora analisando
    'issued',        -- seguradora emitiu, aguarda registro
    'contracted',    -- apólice criada
    'rejected'       -- recusada (cliente ou seguradora)
  ));

-- 2. Adicionar campos de proposta à seguradora
ALTER TABLE quote_requests
  ADD COLUMN IF NOT EXISTS proposal_number TEXT,
  ADD COLUMN IF NOT EXISTS protocol_number TEXT,
  ADD COLUMN IF NOT EXISTS response_deadline DATE,
  ADD COLUMN IF NOT EXISTS rejection_reason TEXT,
  ADD COLUMN IF NOT EXISTS submitted_at TIMESTAMPTZ,
  ADD COLUMN IF NOT EXISTS issued_at TIMESTAMPTZ,
  ADD COLUMN IF NOT EXISTS source TEXT,
  ADD COLUMN IF NOT EXISTS original_lead_id UUID REFERENCES leads(id) ON DELETE SET NULL;

-- 3. Criar deal_stage_history
CREATE TABLE IF NOT EXISTS deal_stage_history (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  deal_id UUID NOT NULL REFERENCES quote_requests(id) ON DELETE CASCADE,
  broker_id UUID NOT NULL REFERENCES brokers(id) ON DELETE CASCADE,
  from_stage TEXT NOT NULL,
  to_stage TEXT NOT NULL,
  note TEXT,
  changed_at TIMESTAMPTZ DEFAULT now()
);

CREATE INDEX IF NOT EXISTS deal_stage_history_deal_id_idx ON deal_stage_history (deal_id);
CREATE INDEX IF NOT EXISTS deal_stage_history_broker_id_idx ON deal_stage_history (broker_id);

ALTER TABLE deal_stage_history ENABLE ROW LEVEL SECURITY;
CREATE POLICY "broker own deal history" ON deal_stage_history
  FOR ALL USING (
    broker_id = (SELECT id FROM brokers WHERE user_id = auth.uid() LIMIT 1)
  );

-- 4. Migrar leads sem quote_request para quote_requests (status prospecting)
INSERT INTO quote_requests (broker_id, client_id, ramo, notes, status, source, original_lead_id, created_at)
SELECT
  l.broker_id,
  l.client_id,
  COALESCE(l.ramo, 'auto'),
  l.name,
  'prospecting',
  'lead_migrado',
  l.id,
  l.created_at
FROM leads l
WHERE NOT EXISTS (
  SELECT 1 FROM quote_requests qr WHERE qr.lead_id = l.id
)
ON CONFLICT DO NOTHING;

-- 5. Migrar dados de proposals para quote_requests
UPDATE quote_requests qr
SET
  proposal_number = p.proposal_number,
  protocol_number = p.protocol_number,
  submitted_at = p.sent_at,
  rejection_reason = CASE WHEN p.status = 'recusada' THEN p.notes ELSE NULL END
FROM proposals p
WHERE p.lead_id = qr.lead_id
  AND qr.status IN ('approved', 'contracted');

-- 6. Índice extra
CREATE INDEX IF NOT EXISTS quote_requests_submitted_at_idx ON quote_requests (broker_id, submitted_at);

-- NOTA: tabela proposals mantida por 1 sprint para rollback safety.
-- Remover em migration futura após Epic 8 estável.
COMMENT ON TABLE proposals IS 'DEPRECATED — dados migrados para quote_requests. Remover após Epic 8.';
```

## Contexto

O pipeline atual é um Kanban simples de leads — ele não sabe nada sobre cotações, propostas ou apólices. Com o Deal Room, o pipeline passa a ser o **cockpit de todas as negociações ativas**, mostrando o estágio real de cada deal — do primeiro contato até a emissão.

As colunas deixam de representar "estágio do lead" e passam a representar **estágio da negociação**. Cada card é um `deal` (que antes era `quote_request` ou `lead`).

---

## Acceptance Criteria

- [x] **AC-1:** `/pipeline` reconstruído com 7 colunas mapeando os stages ativos: Prospecção | Em Cotação | Enviada | Aprovada | Com Seguradora | Em Análise | Emitida
- [x] **AC-2:** Stages `contracted` e `rejected` **não aparecem** no Kanban — deals contratados vão para `/apolices`; rejeitados vão para arquivo acessível via filtro
- [x] **AC-3:** Cada card de deal exibe: nome do cliente, ramo (badge colorido), valor estimado (prêmio do item recomendado ou estimativa), dias no estágio atual, indicador se cliente visualizou o link ("Visto há X min" quando `view_count > 0`)
- [x] **AC-4:** Botão "Novo Deal" no topo do pipeline → modal `NewDealModal` com: cliente (autocomplete), ramo, descrição do objeto, origem (indicação/site/ligação/evento)
- [x] **AC-5:** Arrastar card entre colunas executa `updateDealStage` — se a transição não for permitida, o card volta para a posição original com toast de erro
- [x] **AC-6:** Clicar no card abre o **Deal Workspace** (`/deals/[id]`) em vez de um modal pequeno
- [x] **AC-7:** Badge contador em cada coluna: número de deals + valor total estimado da coluna
- [x] **AC-8:** Coluna "Com Seguradora" e "Em Análise" mostram `response_deadline` no card quando preenchido — vermelho se vencido
- [x] **AC-9:** Coluna "Emitida" tem botão de ação rápida "Registrar Apólice" diretamente no card (abre modal de conversão da 8.5)
- [x] **AC-10:** Filtros no topo: por ramo, por origem, busca por nome de cliente
- [x] **AC-11:** Deal em `prospecting` (lead sem cotação) mostra botão "Iniciar Cotação" que muda stage para `draft` e abre o workspace
- [x] **AC-12:** Supabase Realtime: stage updates de outros dispositivos refletem sem reload

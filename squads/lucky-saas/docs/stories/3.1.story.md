## Story Statement

**Como** Carlos (corretor analógico) ou Roberto (dono da micro-corretora),  
**quero** ver todos os meus alertas de renovação e recuperação num único lugar com urgência visual clara,  
**para que** eu saiba exatamente quais clientes contatar hoje e possa dispensar alertas já tratados.

---

## Context & Background

A Central de Alertas é o hub operacional diário do corretor. Os alertas são gerados automaticamente pela Story 2.1 (renewal_90d, renewal_60d, renewal_30d) e pela Story 1.4 (recovery_reminder). Esta story cria a UI de visualização + dismiss.

**User stories do PRD atendidas:** US-03.1, US-03.2

## Acceptance Criteria

### AC1: Lista de Alertas Pendentes

- [x] Página `/alertas` exibe todos os alertas com `status = 'pending'`
- [x] Alertas agrupados em 3 seções: **Atrasados** (< 0 dias) | **Próximos 7 dias** | **Próximos 30 dias**
- [x] Contador de alertas na seção header mostra total pendente
- [x] Cada alerta exibe: ícone por tipo, título, nome do cliente/lead, ramo (badge verde), data agendada, badge de urgência (dias restantes)

### AC2: Badge de Urgência

- [x] Atrasado (< 0 dias): fundo vermelho `#FEE2E2`, texto `Atrasado`
- [x] Próximos 7 dias: fundo amarelo `#FEF3C7`, texto `Nd` (ex: "5d")
- [x] Futuro (> 7 dias): fundo cinza `#F3F4F6`, texto `Nd`
- [x] Fundo do card muda conforme urgência (vermelho / amarelo / branco)

### AC3: Ações no Alerta

- [x] Botão WhatsApp → `wa.me/55{telefone}` em nova aba (aparece se cliente/lead tem telefone)
- [x] Link "Ver apólice" → `/apolices` (aparece se alerta vinculado a apólice)
- [x] Link "Ver lead" → `/pipeline` (aparece se alerta vinculado a lead)
- [x] Botão dispensar (X) → otimista: remove da lista imediatamente, persiste via Server Action

### AC4: Tab Dispensados

- [x] Tab "Dispensados" lista alertas com `status = 'dismissed'`
- [x] Itens dispensados com opacidade 60%, sem ações — somente referência
- [x] Contagem na tab atualiza conforme alertas dispensados

### AC5: Server Actions

- [x] `dismissAlert(alertId)` — atualiza `status = 'dismissed'`, revalida `/alertas` e `/dashboard`
- [x] `fetchAlerts(status)` — busca alertas com join `policies → clients` e `leads`
- [x] Dismiss otimista via `useTransition` — UI responde imediatamente sem wait de server

### AC6: Ícones por Tipo

- [x] `renewal_*` → ícone `Shield`
- [x] `recovery_reminder` → ícone `RefreshCw`
- [x] Outros → ícone `Bell`

### AC7: PageHeader

- [x] Subtitle dinâmico: "N alerta(s) pendente(s)" ou "Tudo em dia"

## Scope

### IN
- Visualização de alertas pendentes e dispensados
- Dismiss otimista
- Links de ação (WhatsApp, ver apólice, ver lead)
- Agrupamento por urgência

### OUT
- Criação manual de alertas (gerenciado automaticamente pelas stories 2.1 e 1.4)
- Notificações push/email (story 3.2)
- Snooze / reagendar alerta (story 3.3)
- Filtros avançados (story 3.x)

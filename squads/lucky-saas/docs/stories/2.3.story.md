## Story Statement

**Como** Carlos ou Juliana (corretores),  
**quero** fazer upload do PDF de uma apólice existente e ter os campos preenchidos automaticamente,  
**para que** eu não precise digitar dados que já estão no documento e reduza erros de cadastro.

---

## Context & Background

Corretores que estão migrando para o Lucky SaaS têm dezenas de apólices ativas nos concorrentes. Digitar cada uma manualmente é um ponto de fricção enorme. Com extração via IA (Anthropic API + vision), o corretor faz upload do PDF, a IA lê e preenche os campos — o corretor só revisa e confirma.

A migration `20260508000001` já adicionou `policies.anexo_url` para guardar a URL do PDF após upload. Esta story entrega o fluxo completo: upload → extração → pré-preenchimento → confirmação → save.

---

## Acceptance Criteria

- [ ] **AC1 — Botão "Importar PDF" no ApolicaModal**: Novo botão "Importar PDF" no header do modal. Ao clicar, abre file picker aceitando apenas `.pdf`. Tamanho máximo: 10MB.
- [ ] **AC2 — Upload para Supabase Storage**: O arquivo é enviado para o bucket `policy-attachments/{broker_id}/{uuid}.pdf` com acesso privado. Server action `uploadPolicyPDF(file: File)` retorna a URL pública do arquivo.
- [ ] **AC3 — Extração via IA**: Server action `extractPolicyData(pdfUrl: string)` chama a Anthropic API (claude-haiku-4-5) com o PDF convertido para base64. O prompt instrui extração dos campos: nome do cliente, telefone, seguradora, ramo, número da apólice, início/fim vigência, prêmio total, CPF/CNPJ.
- [ ] **AC4 — Pré-preenchimento do formulário**: Os campos extraídos são mapeados para os campos do ApolicaModal. Campos preenchidos pela IA ficam com borda azul-claro para indicar "preenchido automaticamente — confirme".
- [ ] **AC5 — Indicador de confiança**: Cada campo extraído mostra um ícone: ✓ verde (alta confiança) ou ⚠ amarelo (baixa confiança). Campos com baixa confiança ficam highlighted para revisão obrigatória.
- [ ] **AC6 — Loading state**: Durante upload + extração, o modal mostra um skeleton/spinner com mensagem "Lendo apólice com IA... (~10s)". Botões desabilitados durante o processo.
- [ ] **AC7 — Fallback gracioso**: Se a extração falhar (API error, PDF ilegível, etc.), exibe toast de erro e mantém o formulário em branco para preenchimento manual. Nunca bloqueia o fluxo.
- [ ] **AC8 — Salvar URL do anexo**: Ao criar a apólice, o `policies.anexo_url` é preenchido com a URL do PDF no Supabase Storage.
- [ ] **AC9 — Visualizar anexo no perfil**: Na tela de perfil do cliente (`/clientes/[id]`), apólices com `anexo_url` mostram um ícone de PDF clicável que abre o documento.

## Technical Notes

### Supabase Storage

```sql
-- Bucket a criar via Supabase Dashboard:
-- Nome: policy-attachments
-- Tipo: Private (acessado via signed URLs)
-- Política: broker pode upload/download apenas seus próprios arquivos
```

```typescript
// Upload
const { data } = await supabase.storage
  .from('policy-attachments')
  .upload(`${brokerId}/${uuid}.pdf`, file, { contentType: 'application/pdf' })

// Signed URL para enviar à IA (1 hora de validade)
const { data: { signedUrl } } = await supabase.storage
  .from('policy-attachments')
  .createSignedUrl(path, 3600)
```

### Anthropic API

```typescript
// Server action (nunca expor API key no client)
// NEXT_PUBLIC_ANTHROPIC_API_KEY → ANTHROPIC_API_KEY (server only)

import Anthropic from '@anthropic-ai/sdk'

const client = new Anthropic({ apiKey: process.env.ANTHROPIC_API_KEY })

const response = await client.messages.create({
  model: 'claude-haiku-4-5-20251001',
  max_tokens: 1024,
  messages: [{
    role: 'user',
    content: [
      { type: 'document', source: { type: 'url', url: signedUrl } },
      { type: 'text', text: EXTRACTION_PROMPT }
    ]
  }]
})
```

### Prompt de Extração

```
Extraia os seguintes dados desta apólice de seguro brasileira em JSON:
{
  "nome_cliente": string | null,
  "telefone": string | null,
  "cpf_cnpj": string | null,
  "seguradora": string | null,
  "ramo": "auto" | "vida" | "residencial" | "empresarial" | "saude" | "rural" | "viagem" | "outros" | null,
  "numero_apolice": string | null,
  "start_date": "YYYY-MM-DD" | null,
  "end_date": "YYYY-MM-DD" | null,
  "premium_total": number | null,
  "confidence": { [campo]: "high" | "low" }
}
Retorne APENAS o JSON, sem explicações.
```

## Story Statement

**Como** Carlos (corretor analógico) ou Juliana (corretora escalando),  
**quero** cadastrar leads manualmente e visualizá-los num Kanban com 5 colunas de status,  
**para que** eu nunca perca um contato e saiba exatamente em que fase está cada negociação.

---

## Context & Background

Este é o primeiro módulo de negócio do Lucky SaaS. O Pipeline é o ponto de entrada de todos os leads — seja cadastro manual, seja importação via ManyChat (story 1.2). A visão Kanban é o centro de trabalho diário do corretor para acompanhar negociações.

**User stories do PRD atendidas:** US-01.1, US-01.3, US-01.5

## Scope

### IN
- Kanban board (5 colunas fixas)
- CRUD de leads (criar + mover + anotar)
- Drawer de detalhe com histórico

### OUT

- Edição/exclusão de lead (story 1.3)

## Technical Notes

### Schema relevante

```sql
-- tabela leads (do architecture.md)
leads (
  id uuid,
  broker_id uuid,       -- RLS filtra por isso
  name text,
  phone text,
  email text,
  ramo text,            -- enum: auto | vida | saude | residencial | empresarial | viagem | consorcio
  source text,          -- enum: instagram | indicacao | site | whatsapp | outro
  status text,          -- enum: novo | cotacao | negociacao | fechado | perdido
  notes text,
  created_at timestamptz,
  last_activity_at timestamptz  -- atualizado a cada evento em lead_activities
)

lead_activities (
  id uuid,
  lead_id uuid,
  broker_id uuid,       -- RLS
  type text,            -- status_change | note
  content text,         -- para notes: texto | para status_change: novo status
  previous_status text, -- só para status_change
  created_at timestamptz
)
```

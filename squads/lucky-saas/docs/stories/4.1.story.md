## Story Statement

**Como** Carlos (corretor analógico) ou Roberto (dono da micro-corretora),  
**quero** ver um dashboard financeiro com total de comissões esperadas, breakdown por ramo e por seguradora, e as renovações próximas com seus valores,  
**para que** eu tenha visibilidade financeira da minha carteira e possa priorizar as renovações mais valiosas.

---

## Context & Background

O módulo Financeiro é o "relatório financeiro" do corretor — mostra o que ele tem em carteira e o que vai receber conforme as renovações chegarem. Os dados vêm da tabela `policies` (commission_expected, premium_total, commission_pct).

**User stories do PRD atendidas:** US-05.1, US-05.2

## Acceptance Criteria

### AC1: Métricas Principais

- [x] Card: **Carteira Ativa** — contagem de apólices ativas vigentes
- [x] Card: **Prêmios Ativos** — soma de `premium_total` das apólices ativas
- [x] Card: **Comissões Esperadas** — soma de `commission_expected` das apólices ativas
- [x] Card: **Próximos 30d** — soma de `commission_expected` das apólices que vencem em 30 dias

### AC2: Breakdown por Ramo

- [x] Lista agrupada por ramo: nome do ramo, count de apólices, comissão total
- [x] Barra de progresso horizontal mostrando % da comissão total
- [x] Ordenado do maior para o menor em comissão

### AC3: Próximas Renovações (90 dias)

- [x] Lista das apólices que vencem nos próximos 90 dias
- [x] Cada item: nome do cliente, ramo, seguradora, data de vencimento, comissão esperada, badge de urgência (Nd)
- [x] Badge colorido: ≤7d vermelho, ≤30d amarelo, >30d cinza
- [x] Máximo 8 itens exibidos, com "+N mais" se houver mais

### AC4: Tabela por Seguradora

- [x] Tabela com colunas: Seguradora | Ramo | Apólices | Prêmio Total | Comissão Esp. | % Méd.
- [x] Ordenada por comissão esperada desc
- [x] Valores monetários em `font-mono`, alinhados consistentemente

### AC5: PageHeader

- [x] Subtitle: "N apólice(s) ativa(s) · R$ X em comissões"

### AC6: Qualidade

- [x] Server component (zero JS client para fetch inicial)
- [x] Todos os valores em `pt-BR` locale (R$ 1.800,00)
- [x] Empty states quando não há apólices cadastradas
- [x] Sem hardcode de cores ou fontes

## Scope

### IN
- Dashboard de visão financeira baseado em `commission_expected`

### OUT

## Technical Notes

### Campos usados da tabela `policies`

```
commission_expected   — comissão calculada (pct × premium)
commission_pct        — percentual de comissão
premium_total         — prêmio total da apólice
ramo                  — ramo do seguro
seguradora            — nome da seguradora
end_date              — vencimento (filtra próximas renovações)
status                — filtra apenas 'ativa'
```

## Story Statement

**Como** Carlos (corretor analógico) ou Roberto (dono da micro-corretora),  
**quero** cadastrar e gerenciar todas as apólices da minha carteira com alertas automáticos de vencimento,  
**para que** eu nunca perca uma renovação e tenha visão completa de toda a carteira num único lugar.

---

## Context & Background

Este é o módulo central do Lucky SaaS — tudo orbita em torno da carteira de apólices. Sem apólices cadastradas, não há alertas, não há cross-sell, não há financeiro. A implementação deve suportar carteiras de até 5.000 apólices sem degradação (NFR-P03).

**User stories do PRD atendidas:** US-02.1, US-02.2, US-02.3, US-02.4

**Referências:**
- `squads/lucky-saas/docs/front-end-spec.md` seção 6 (Apólices)
- `squads/lucky-saas/app/DESIGN.md` (tabela, badges de status, drawer)
- `squads/lucky-saas/docs/architecture.md` (tabelas `policies`, `clients`, `alerts`)

## Acceptance Criteria

### AC1: Lista de Apólices (`/apolices`)

- [x] Tabela com colunas: Cliente | Ramo | Seguradora | Vigência | Prêmio | Comissão | Status | Ações
- [x] Paginação server-side: 25 por página, contador "Mostrando 1-25 de 347"
- [x] Valores monetários em JetBrains Mono, alinhados à direita
- [x] Linha com vencimento < 30 dias: fundo `#FFFBEB`
- [x] Linha vencida (fim de vigência < hoje): fundo `#FFF5F5`
- [x] Coluna Ações: menu de 3 pontos com opções Editar | Ver histórico | Arquivar
- [x] Skeleton loading enquanto dados carregam

### AC2: Filtros e Tabs

- [x] Tabs: Todas | Ativas | Vencendo em 30d | Vencidas | Arquivadas
- [x] Filtros: Ramo (select) | Seguradora (select) | Status (select) | Período (date range)
- [x] Input de busca filtra por nome do cliente ou número da apólice
- [x] URL atualiza com query params ao aplicar filtros (permite bookmark/share de filtros)
- [x] Contagem de resultados atualiza conforme filtros

### AC3: Cadastro de Apólice (Modal)

- [x] Botão "Nova Apólice" no page header abre modal (Dialog shadcn, max-width 640px)
- [x] **Seção 1 — Apólice:**
  - Cliente* (autocomplete dos clientes cadastrados, com opção "+ Criar novo cliente" inline)
  - Ramo* (select com os ramos disponíveis do corretor)
  - Seguradora* (input livre)
  - Número da apólice (opcional)
- [x] **Seção 2 — Vigência e Financeiro:**
  - Data início vigência* (date picker)
  - Data fim vigência* (date picker — não pode ser anterior ao início)
  - Prêmio total (R$)* — input numérico com formatação automática
  - Forma de pagamento (select: Anual | Semestral | Mensal)
  - % de comissão* — input numérico (0-100)
  - Comissão calculada (readonly — atualiza automaticamente: prêmio × % ÷ 100)
- [x] **Seção 3 — Dados do Ramo** (renderiza campos condicionais baseado no ramo selecionado):
  - Auto: placa, modelo/ano
  - Vida: CPF do segurado, capital segurado
  - Residencial: endereço (textarea)
  - Empresarial: CNPJ, ramo de atividade
  - Outros ramos: sem campos extras
- [x] Ao salvar: apólice criada no banco, alertas de 90/60/30 dias agendados automaticamente em `alerts`
- [x] Se número de apólice já existir para mesma seguradora: warning dialog (não bloqueia, permite continuar)

### AC4: Alertas de Vencimento — Criação Automática

- [x] Ao criar ou editar a data de fim de vigência: alertas são calculados e inseridos/atualizados em `alerts`
- [x] Tipos de alerta criados: `renewal_90d`, `renewal_60d`, `renewal_30d`
- [x] `scheduled_for` = `end_date - N days` para cada tipo
- [x] Se `scheduled_for` já passou (apólice cadastrada com vencimento próximo ou já passado): alerta criado com `status = 'pending'` para aparecer imediatamente no dashboard

### AC5: Template de Mensagem de Renovação

- [x] Na lista e no detalhe: botão "Contatar" ao lado de apólices com vencimento ≤ 30 dias
- [x] Ao clicar: modal com template pré-preenchido usando dados da apólice:
  ```
  "Oi {nome_cliente}! Tudo bem? Seu seguro {ramo} está vencendo em {dias} dias 
  ({data_vencimento}). Posso te ajudar a renovar? Abraços, {nome_corretor}"
  ```
- [x] Template editável antes de abrir o WhatsApp
- [x] Botão "Abrir WhatsApp" → `window.open('https://wa.me/55{telefone}?text={encoded_msg}')`
- [x] Ao enviar: evento registrado no histórico da apólice

### AC6: Perfil do Cliente (`/clientes/[id]`)

- [x] Página do cliente exibe: dados cadastrais, total de prêmios, total de comissões esperadas
- [x] Lista todas as apólices do cliente (ativas, arquivadas separadas)
- [x] Timeline de contatos (eventos da apólice relacionados ao cliente)
- [x] Botão "Nova Apólice" já pre-seleciona esse cliente no formulário

### AC7: Autocomplete de Clientes + Criação Inline

- [x] No campo "Cliente" do formulário: busca em `/api/clients?q={termo}` com debounce 300ms
- [x] Lista dropdown com max 8 resultados, ícone de criação no final se nenhum resultado
- [x] "+ Criar cliente: [nome digitado]" cria cliente com dados mínimos (nome + telefone via mini-form inline no modal)
- [x] Novo cliente fica selecionado automaticamente após criação

### AC8: Qualidade

- [x] Tabela paginada renderiza 5.000 apólices sem degradação visual (paginação server-side garante isso)
- [x] Valores monetários sempre formatados como `R$ 1.800,00` (locale pt-BR)
- [x] Datas sempre formatadas como `dd/MM/yyyy`
- [x] `npm run build` passa sem erros após esta story

## Technical Notes

### Schema relevante

```sql
-- tabela policies (do architecture.md)
policies (
  id uuid,
  broker_id uuid,         -- RLS
  client_id uuid,         -- FK clients
  ramo text,
  seguradora text,
  policy_number text,
  start_date date,
  end_date date,
  premio_total numeric,
  payment_type text,       -- anual | semestral | mensal
  comissao_percent numeric,
  comissao_valor numeric,  -- generated: premio_total * comissao_percent / 100
  status text,             -- ativa | vencida | cancelada | arquivada
  extra_data jsonb,        -- dados específicos do ramo (placa, CPF, etc.)
  created_at timestamptz
)

-- tabela clients
clients (
  id uuid,
  broker_id uuid,         -- RLS
  name text,
  phone text,
  email text,
  cpf text,
  birth_date date,
  created_at timestamptz
)

-- tabela alerts
alerts (
  id uuid,
  broker_id uuid,         -- RLS
  policy_id uuid,
  type text,              -- renewal_90d | renewal_60d | renewal_30d | birthday | etc.
  scheduled_for date,
  status text,            -- pending | sent | dismissed
  sent_at timestamptz
)
```

### Server Actions

```
src/app/(dashboard)/apolices/actions.ts
  - createPolicy(data)
  - updatePolicy(id, data)
  - archivePolicy(id)
  - scheduleRenewalAlerts(policyId, endDate)
  - markAlertSent(alertId)
```

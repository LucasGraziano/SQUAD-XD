## Story Statement

**Como** equipe Lucky SaaS,  
**queremos** uma landing page que converta corretores SUSEP visitantes em cadastros,  
**para que** o produto tenha aquisição orgânica e possamos validar a proposta de valor antes de investir em tráfego pago.

---

## Context & Background

O app está funcional (Pipeline, Apólices, Clientes). Falta a porta de entrada pública.
A landing page vive em `/` (public route) enquanto o dashboard vive em `/dashboard`.
O design deve seguir `DESIGN.md` em `squads/lucky-saas/app/DESIGN.md` fielmente.
A página deve ser estática (Next.js `export`) ou Server Component para máximo SEO.

**Persona primária:** Carlos (corretor analógico, ~40 anos, usa WhatsApp, desconfia de "sistema")
**Persona secundária:** Juliana (corretora escalando, ~30 anos, quer controle e profissionalismo)

---

## Acceptance Criteria

- [x] **AC1 — Rota pública**: `/` renderiza a landing page sem autenticação. Usuários logados são redirecionados para `/dashboard`.
- [x] **AC2 — Seção Hero**: Headline impactante (problema-first), subheadline com benefício quantificado, CTA principal "Começar grátis" → `/signup`, visual do produto (screenshot ou mockup fiel ao app real).
- [x] **AC3 — Seção Features**: 3 cards com ícone + título + 1 linha: "Pipeline de leads", "Alertas de renovação", "Controle de comissões".
- [x] **AC4 — Seção Prova Social**: Contador de corretores (pode ser placeholder honesto "seja um dos primeiros") + 1 depoimento ou caso de uso.
- [x] **AC5 — Seção Como Funciona**: 3 passos visuais: Cadastre → Importe/cadastre clientes → Acompanhe renovações.
- [x] **AC6 — Seção Pricing**: Plano gratuito + plano pago. Simples, sem tabela complexa. CTA em cada plano.
- [x] **AC7 — CTA Final**: Repetição do CTA hero com variação de copy.
- [x] **AC8 — Header sticky**: Logo + nav links (Features, Preços) + botão "Entrar" (→ /login) + botão "Começar grátis" (→ /signup).
- [x] **AC9 — Footer**: Logo + links úteis + "© 2026 Lucky SaaS" + LGPD/Política de privacidade link.
- [x] **AC10 — Responsivo**: Mobile-first. Funciona corretamente em 375px, 768px e 1280px.
- [x] **AC11 — Performance**: LCP < 2.5s, sem layout shift visível. Imagens otimizadas com `next/image`.
- [x] **AC12 — Design fiel ao DESIGN.md**: Cores, tipografia, espaçamentos e componentes exatamente conforme `squads/lucky-saas/app/DESIGN.md`. Zero desvio de design.

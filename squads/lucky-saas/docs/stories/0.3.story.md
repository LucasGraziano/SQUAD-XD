## Story Statement

**Como** equipe Lucky SaaS,  
**queremos** integrar Stripe para processar assinaturas do Plano Pro,  
**para que** o produto seja monetizável e a landing page converta em receita real.

---

## Context & Background

A landing page (Story 0.2) já exibe o Plano Pro (R$49/mês) com CTA → `/signup?plan=pro`.
Hoje esse link cria uma conta gratuita sem nenhuma distinção de plano.

Esta story fecha o loop: cadastro → checkout Stripe → ativação do plano → acesso às features Pro.

**Estratégia de billing:**
- Free plan: sem cartão, limites aplicados via código (50 clientes, 50 apólices)
- Pro plan: Stripe Checkout → webhook ativa `subscription_status = 'active'` na tabela `brokers`
- Trial: 14 dias gratuitos ao selecionar Pro no signup

---

## Acceptance Criteria

- [ ] **AC1 — Tabela `brokers` com campos de billing**: `subscription_status` (free|trialing|active|canceled|past_due), `stripe_customer_id`, `stripe_subscription_id`, `trial_ends_at`, `subscription_ends_at`. Migration aplicável.

- [ ] **AC2 — Signup com plano**: `/signup?plan=pro` inicia trial de 14 dias automaticamente ao criar conta. `/signup` sem param = free.

- [ ] **AC3 — Stripe Checkout**: Rota `/api/billing/checkout` cria sessão Stripe Checkout com price_id correto, `trial_period_days: 14`, sucesso → `/dashboard?upgraded=true`.

- [ ] **AC4 — Webhook Stripe**: Rota `/api/billing/webhook` processa `checkout.session.completed`, `customer.subscription.updated`, `customer.subscription.deleted` → atualiza `brokers` via Supabase service role.

- [ ] **AC5 — Portal do cliente**: Rota `/api/billing/portal` cria sessão do Stripe Customer Portal → usuário gerencia cartão e cancela sem falar com suporte.

- [ ] **AC6 — Limites Free enforcement**: Server actions de `createClient` e `createPolicy` verificam quota (50 cada). Se exceder → retornam `{ error: 'QUOTA_EXCEEDED' }`. Frontend exibe modal de upgrade.

- [ ] **AC7 — Página `/configuracoes/billing`**: Exibe plano atual, status, data de renovação/trial. Botão "Fazer upgrade" → AC3. Botão "Gerenciar assinatura" → AC5. Badge do plano visível na sidebar.

- [ ] **AC8 — Banner de trial**: Dashboard exibe banner amarelo "X dias restantes no trial Pro" enquanto `subscription_status = 'trialing'`. Botão "Adicionar cartão" no banner.

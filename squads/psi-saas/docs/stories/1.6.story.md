# Story 1.6 — RLS Hardening: WITH CHECK Explícito em Todas as Tabelas

**Epic:** 001 — Foundation
**Status:** Done
**Agent:** @data-engineer (Dara)

## Description

As políticas RLS da migration `001_initial_schema.sql` usam a clause `USING` sem `FOR` explícito, fazendo o Postgres derivar o `WITH CHECK` automaticamente do `USING`. Embora funcional, isso é um risco de segurança e auditoria:

1. **Sem separação explícita** — impossível auditar intenção da política por operação (SELECT vs INSERT vs UPDATE vs DELETE)
2. **LGPD Art. 46** exige medidas técnicas adequadas; políticas implícitas não passam em auditorias formais
3. **Policies futuras** que adicionem `FOR SELECT` podem remover acidentalmente a proteção de write das `ALL` policies existentes
4. **CFP Res. 001/2009** — dados de prontuário (session_notes, sessions) exigem controle de escrita auditável

O fix cria a migration `002_rls_with_check.sql` que dropa cada política `ALL` e as reconstrói em 4 políticas granulares por tabela: `FOR SELECT`, `FOR INSERT WITH CHECK`, `FOR UPDATE USING + WITH CHECK` e `FOR DELETE`.

## Dev Notes

### Padrão por tabela

Cada tabela que usa `current_psychologist_id()` deve seguir este padrão:

```sql
-- Dropar policy all-in-one existente
DROP POLICY IF EXISTS "psy_own_{name}" ON {table};

-- SELECT: usa USING
CREATE POLICY "psy_select_{name}" ON {table}
  FOR SELECT USING (psychologist_id = current_psychologist_id());

-- INSERT: usa WITH CHECK (sem USING — inaplicável)
CREATE POLICY "psy_insert_{name}" ON {table}
  FOR INSERT WITH CHECK (psychologist_id = current_psychologist_id());

-- UPDATE: usa USING (quais rows) + WITH CHECK (novos valores)
CREATE POLICY "psy_update_{name}" ON {table}
  FOR UPDATE
  USING (psychologist_id = current_psychologist_id())
  WITH CHECK (psychologist_id = current_psychologist_id());

-- DELETE: usa USING
CREATE POLICY "psy_delete_{name}" ON {table}
  FOR DELETE USING (psychologist_id = current_psychologist_id());
```

### Tabela `psychologists` (user_id, não psychologist_id)

```sql
DROP POLICY IF EXISTS "psy_own_row" ON psychologists;

CREATE POLICY "psy_select_own_row" ON psychologists
  FOR SELECT USING (user_id = auth.uid());

CREATE POLICY "psy_insert_own_row" ON psychologists
  FOR INSERT WITH CHECK (user_id = auth.uid());

CREATE POLICY "psy_update_own_row" ON psychologists
  FOR UPDATE
  USING (user_id = auth.uid())
  WITH CHECK (user_id = auth.uid());

CREATE POLICY "psy_delete_own_row" ON psychologists
  FOR DELETE USING (user_id = auth.uid());
```

### Tabela `audit_log` (append-only)

```sql
-- audit_log já tem policies separadas, apenas reforçar:
DROP POLICY IF EXISTS "psy_own_audit_read" ON audit_log;
DROP POLICY IF EXISTS "psy_own_audit_insert" ON audit_log;

CREATE POLICY "psy_select_audit" ON audit_log
  FOR SELECT USING (psychologist_id = current_psychologist_id());

CREATE POLICY "psy_insert_audit" ON audit_log
  FOR INSERT WITH CHECK (psychologist_id = current_psychologist_id());

-- Sem UPDATE e DELETE policies → append-only reforçado por RLS + rules
```

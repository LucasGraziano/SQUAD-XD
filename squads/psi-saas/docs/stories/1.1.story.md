## Notes

- Criptografia: PBKDF2 200k iterações → AES-256-GCM. Salt e IV únicos por nota. Server nunca vê plaintext.
- RLS: todas as tabelas protegidas. `current_psychologist_id()` função helper SECURITY DEFINER.
- Imutabilidade: trigger `lock_note_after_24h` — CFP Res. 001/2009 compliant.
- Audit log: append-only via `CREATE RULE no_update/delete_audit`.
- Dashboard: Server Component, busca dados em paralelo com `Promise.all`.

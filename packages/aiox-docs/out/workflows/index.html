Workflows & Tasks

O AIOX orquestra desenvolvimento através de 10 workflows compostos por 207 tasks estruturadas. Greenfield para projetos novos, Brownfield para sistemas existentes, e Meta-workflows para o ciclo de desenvolvimento contínuo.

Greenfield vs Brownfield — Qual usar?

🌱

Greenfield

Novo

Projeto do zero, sem código existente. Começa com PRD → Arquitetura → Schema → Stories → Dev.

  • → Novo SaaS ou app completo
  • → Nova API ou microservice
  • → Nova UI ou landing page
🏚️

Brownfield

Existente

Sistema existente com código legado. Sempre começa com assessment (Discovery) antes de modificar.

  • → Entrar em projeto herdado
  • → Refactor / migração de stack
  • → Redesign de UI legada
🔄

Meta-Workflows

Sempre

Usados em todo projeto, independente de ser novo ou legado. SDC, QA Loop e Spec Pipeline são universais.

  • → SDC: cada story implementada
  • → QA Loop: problemas iterativos
  • → Spec Pipeline: features complexas

Story Development Cycle (SDC)

Full 4-phase workflow for all development work. Create → Validate → Implement → QA Review.

PRIMARY
4 fases|Usar quando: New story from epic, any development task
1
🌊@sm→create-next-story.md

Create story from PRD/epic

Output: story.md
↓
2
🎯@po→validate-next-story.md

10-point validation checklist

Output: GO/NO-GO verdict
↓
3
💻@dev→dev-develop-story.md

Implement with CodeRabbit self-healing

Output: Working code
↓
4
✅@qa→qa-gate.md

7-point quality gate

Output: PASS/FAIL verdict

QA Loop

Automated review-fix cycle. @qa reviews → verdict → @dev fixes → re-review (max 5 iterations).

ITERATIVE
3 fases|Usar quando: QA found issues, need iteration
1
✅@qa→qa-review

Review code quality and criteria

Output: APPROVE/REJECT/BLOCKED
↓
2
💻@dev→dev-apply-qa-fixes

Apply fixes from QA feedback

Output: Fixed code
↓
3
✅@qa→qa-re-review

Re-review after fixes (max 5x)

Output: Final verdict

Spec Pipeline

Transform informal requirements into executable spec. 6 phases with complexity-based phase skipping.

PRE-IMPLEMENTATION
6 fases|Usar quando: Complex feature needs spec before coding
1
📋@pm→spec-gather

Gather requirements (never skip)

Output: requirements.json
↓
2
📐@architect→spec-assess

Assess complexity (5 dimensions, score 1-25)

Output: complexity.json
↓
3
🔍@analyst→spec-research

Research dependencies (skip if SIMPLE)

Output: research.json
↓
4
📋@pm→spec-write

Write specification (never skip)

Output: spec.md
↓
5
✅@qa→spec-critique

Critique spec (APPROVED/NEEDS_REVISION/BLOCKED)

Output: critique.json
↓
6
📐@architect→spec-plan

Implementation plan (if APPROVED)

Output: implementation.yaml

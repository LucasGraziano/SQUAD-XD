Ciclo de Vida da Memória

📝

1. Criação

  • -Agente observa padrões durante a sessão
  • -Hook agent_memory_persister roda no SessionEnd
  • -Append com timestamp + progresso (%) + files
  • -Max 200 linhas (header + 175 entries recentes)
🔍

2. Recall

  • -Agente ativa → lê MEMORY.md via imports
  • -Synapse calcula bracket de contexto
  • -MemoryBridge filtra por sector preferences
  • -Injeta via <synapse-rules> XML
♻️

3. Promoção

  • -Padrão visto em 3+ agents
  • -Move para Promotion Candidates
  • -Elevado a CLAUDE.md ou .claude/rules/
  • -Padrões obsoletos → Archived (com data)

Bracket System

O sistema adapta a quantidade de memória injetada baseado no contexto restante da conversa. Quanto menos contexto disponível, mais memória é carregada.

FRESH
60-100%
Max 0 tokensskip

Nenhuma memória injetada — contexto disponível é suficiente

MODERATE
40-60%
Max 50 tokens

Layer 1: Metadata — lembretes breves das memórias mais relevantes

DEPLETED
25-40%
Max 200 tokens

Layer 2: Chunks — blocos de conhecimento expandidos para reforço

CRITICAL
0-25%
Max 1000 tokens

Layer 3: Full Content — memória completa + alerta de handoff

Mega Brain Knowledge Store

O Mega Brain armazena conhecimento em formato YAML estruturado com 5 camadas DNA por expert. Cada entrada rastreia evidências, implicações e peso de confiança.

Estrutura YAML (por DNA layer)

idID único (FIL-AH-001)
nomeNome da filosofia/modelo/etc
declaracaoDeclaração completa com contexto
evidenciasCitações diretas com chunk_id e source_id
implicacoesfazer: [...] + evitar: [...]
conflitos_potenciaisConflitos com outros princípios
dominiosTags de domínio (sales, scaling, etc)
pesoConfiança 0.0-1.0 baseada em evidências

Synapse Memory Engine

Motor de injeção de memória do AIOX. Coordena bracket detection, memory retrieval e context injection com fail-open architecture.

Location.aiox-core/core/synapse/
Cache Key{agentId}-{bracket}
Timeout15ms
Fail ModeFail-open (returns empty array)

Retrieval Flow

1
Synapse Engine

Recebe request de ativação do agente

2
calculateBracket()

Calcula % de contexto restante → determina bracket

3
MemoryBridge.getMemoryHints()

Consulta agentId + bracket + tokenBudget

4
SynapseMemoryProvider

Filtra por sector preferences do agente

5
MemoryLoader.queryMemories()

Busca nas MEMORY.md com limite de tokens

6
formatSynapseRules()

Formata como <synapse-rules> XML e injeta no contexto
